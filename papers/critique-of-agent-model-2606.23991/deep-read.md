# 🧠 Critique of Agent Model — 深度解读

> **Critique of Agent Model** | arxiv:2606.23991 | 2026-06-15
>
> **作者：** Eric Xing (MBZUAI/CMU), Mingkai Deng, Jinyu Hou
>
> **论文类型：** Position Paper（立场论文）— 不是实验报告，而是概念性重构

---

## 📋 概述

这是一篇**立场论文**——不做实验，而是回到最基本的问题：「智能体（agent）到底是什么？真正的能动性（agency）由什么构成？」

核心论点：

> "Agentic ≠ Agentive"。多数当前系统是 **agentic**（通过外部 orchestration 完成任务的系统），但远未达到 **agentive**（具备真正内生的、持续的能力）。

---

## 🏗️ 核心框架：Agentic vs. Agentive

论文最重要的分析贡献是区分了两个关键概念：

| | **Agentic 系统** | **Agentive 系统** |
|--|----------------|-----------------|
| **定义** | 通过精心设计的工具和 workbox 完成任务的系统 | 内在地拥有自主行动能力的系统 |
| **能力来源** | 工程化编排（外部 scaffold） | 系统自身内生的决策和学习 |
| **典型例子** | Claude Code, Codex, SWE-agent | 人类、生物智能体、科幻片的 replicants |
| **目标来源** | 用户指令/prompt | 内部持久目标 |
| **身份** | 静态（system prompt） | 动态演变 |
| **决策** | 固定流程 | 内省式模拟推理 |
| **学习** | 外部安排的训练周期 | 自主从经验中学习 |
| **安全** | 可控但脆 | 需要审计能力 |

**关键论断：** 当前 AI 系统多数是 agentic 而非 agentive——它们的 competence 存在于 workflow 和 harness 中，而非模型自身。

---

## 🔬 五维分析框架

论文沿五个维度分析能动性，每个维度都是一个谱系：

### 维度 1：目标（Goal）

| | 外部指定 | ↔ | 内部持久 |
|--|---------|----|---------|
| | 每次交互由用户指令提供 | | 系统拥有持久的自身目标 |
| | prompt 消失 → 目标消失 | | 目标分解 → 子目标序列 |
| **典型：** 当前所有 LLM agent | | **理想：** GIC 架构 |

### 维度 2：身份（Identity）

| | 静态 | ↔ | 演变 |
|--|------|----|------|
| | system prompt 固定 | | 身份随经验更新 |
| | 配置文件中定义 | | 自适应新角色/能力 |

### 维度 3：决策（Decision-making）

| | 固定流程 | ↔ | 模拟推理 |
|--|---------|----|---------|
| | ReAct → Tool → Response | | System I（快速反应）+ System II（慢速模拟推理）|
| | 可预测但僵化 | | 灵活但复杂 |

### 维度 4：自我调节（Self-regulation）

| | 无 | ↔ | 学会何时如何思考 |
|--|----|----|--------------|
| | 每个请求用相同推理深度 | | 紧急情况→快速反应，复杂问题→深入规划 |
| | 浪费算力 | | 效率+适应 |

### 维度 5：学习（Learning）

| | 外部安排 | ↔ | 自我导向 |
|--|---------|----|---------|
| | 人工标注→训练→部署| | 从真实+模拟经验自主学习 |
| | 固定周期 | | 持续自适应 |

## 🏛️ GIC 架构（Goal-Identity-Configurator）

论文的核心理论贡献：提出一个通往 agentive 系统的具体架构蓝图。

**三条核心设计原则：**

### 原则 1：Agent Model ≠ World Model（严格分离）

> Agent Model = 决定「做什么」(p_π(a|s,g,i))
> World Model = 预测「会发生什么」(p_μ(s'|s,a))

论文严厉批评了将两者合并的近期趋势（Ye et al., Li, NVIDIA），指出合并会混淆奖励驱动的行动选择与保真度驱动的状态预测，**同时削弱规划和模拟的可靠性**。

### 原则 2：GIC 三组件

```
                 ┌─────────────────┐
                 │  Goal Module     │ ← 持久目标 + 分层分解
                 │  (元认知层)      │
                 └────────┬────────┘
                          ↓
                 ┌─────────────────┐
                 │  Identity Module │ ← 自我模型：能力/约束/关系
                 │  (自我层)        │    随时间演变
                 └────────┬────────┘
                          ↓
                 ┌─────────────────┐
                 │  Configurator   │ ← 三系统决策+自我调节
                 │  (执行层)        │    System I/II/III
                 └─────────────────┘
```

### 原则 3：三系统智力（System I / II / III）

| 系统 | 类比 | 功能 | 速度 | 适合场景 |
|------|------|------|------|---------|
| **System I** | 本能/直觉 | 快速反应行动 | 毫秒 | 紧急、常规任务 |
| **System II** | 深思熟虑 | 基于世界模型的模拟推理 | 秒-分 | 复杂规划、多步推理 |
| **System III** | 元认知 | 配置何时用 I 还是 II | 持续 | 自我调节、资源配置 |

## 🎯 核心洞见

### 1. "agent" 这个词已经失去意义

论文开门见山：从带 for 循环的工具调用到机器超智能，"agent" 什么都覆盖=什么都不覆盖。大多数现在叫"agent"的系统实际上是精心设计的软件 pipeline，不包含任何能动性。

### 2. 能力在 harness 里，不在模型里

论文指出当前多数 AI 系统的 **competence 存在于工程化流程中**，而非模型自身。如果去掉 ReAct loop、tool definitions、system prompt，剩下的 LLM 本身无法自主完成任务。这是 agentic 与 agentive 最本质的差别。

### 3. 笛卡尔哲学的现代回响

"Cogito, ergo sum"——论文引用了从亚里士多德到笛卡尔的哲学传统，将 agency 放置在独立思考和自由意志的基础上。这不是偶然的学究行为，而是表明论文认为真正的 AI agent 需要在某种意义上是"有思想的"。

### 4. 从清单到蓝图

论文不满足于批评，而是提供了 GIC 架构作为建设性方案。虽然 GIC 还是一个概念架构（论文明确表示没有实现），但它为未来工作提供了清晰的目标和方向。

### 5. 生物智能体的层级

```
哲学层（好奇心、自我反思、目标形成）
     ↑
社会层（协调、竞争、合作）
     ↑
物理层（导航、操控）
     ↑
语言层（读写、编码）
```

人类单一认知架构支持所有这些层级——当前 AI 系统用特定编排为每一层分别设计。

## ⚠️ 局限

1. **纯 position paper**：没有实验、没有实现、没有代码
2. **GIC 架构未实现**：概念蓝图，可行性有待验证
3. **可能过于理想化**：真正 agentive 系统可能在计算上不可行或不需要
4. **对当前系统的价值不明**：如果一个 agentic 系统已经足够好，为什么需要 agentive？
5. **世界模型训练成本未知**：单独训练一个世界模型的成本可能远超收益

## 💡 启示

1. **命名战争刚刚开始**："agentic vs. agentive" 的区分可能会像 "strong AI vs. weak AI" 一样进入常规语汇
2. **基础设施级 insight**：Agent Model 与 World Model 的严格分离可能是未来自主系统的关键设计决策
3. **当前 agent 的"天花板"**：外部 scaffold 模式可能在复杂度上有天花板，论文暗示我们需要新的范式
4. **安全性讨论**：agentive 系统需要全新的审计和控制框架——更强大的能力也需要更严格的可控性设计
5. **从 CMU 到 MBZUAI 的传承**：Eric Xing 在基础模型和 AI 理论方面的深度思考延续了这一线工作
