# 🔬 NatureBench — 深度解读

> **NatureBench: Are Coding Agents Ready for Real Scientific Discovery?** | arxiv:2606.24530 | 2026-06-26
>
> **作者：** PKU + Microsoft Research Asia + Tsinghua

---

## 📋 概述

SWE-bench 被广泛用于评估 coding agent 的能力，但有一个根本问题：**写 issue 修复 ≈ 真实科研吗？**

NatureBench 的答案是：不。真实科研涉及**全新的任务、未见过的方法、多轮的试错迭代和严谨的实证验证**。论文构建了一个 52 个科研题目的基准，测试 coding agent 是否能自己规划和执行实验。

---

## 🏗️ NatureBench 设计

### 52 个科研题目

论文从 **Nature 论文**中提取核心实验，将其转化为 coding tasks。每个题目有三个版本（逐步增加难度）：

| 难度 | 给 Agent 的信息 | 示例 |
|------|---------------|------|
| **Easy** | 完整的方法描述 + 数据集 | 直接实现 |
| **Medium** | 方法描述有缺失，需要推理 | 你猜这部分怎么做 |
| **Hard** | 只有题目 + 论文标题 | 完全自规划 |

覆盖 10 个科研领域：

```
├── 计算生物学（8 题）
├── 机器学习（7 题）
├── 统计物理（6 题）
├── 量子化学（5 题）
├── 计算流体力学（5 题）
├── 网络科学（5 题）
├── 气候建模（4 题）
├── 地震学（4 题）
├── 生物信息学（4 题）
└── 社会计算（4 题）
```

### 自动验证器

每个题目都有**程序化验证器**——自动化检查 Agent 的输出是否符合 Nature 论文中的结果。

验证器类型：
- **数值验证**（误差在 5% 内匹配论文数值结果）
- **图表验证**（生成的可视化是否与论文一致）
- **代码结构验证**（是否使用了正确的算法框架）
- **逻辑验证**（中间的推理步骤是否合理）

### 评估指标

- **Pass@1:** 单次尝试成功率
- **Pass@5:** 5 次尝试成功率
- **Multi-pass:** 多次尝试+迭代修复成功率
- **Settlement time:** 达到验证器标准的中位数时间

## 🔬 实验结果

### 主要结果

| Coding Agent | Pass@1 (Easy) | Pass@1 (Medium) | Pass@1 (Hard) |
|-------------|:-----------:|:-------------:|:-----------:|
| **Claude Code** (最佳) | **52%** | **31%** | **15%** |
| GPT-4o Code Agent | 48% | 27% | 12% |
| GPT-4o-mini | 35% | 18% | 8% |
| DeepSeek-Coder-V2 | 40% | 22% | 10% |
| Codestral | 38% | 20% | 9% |
| Qwen2.5-Coder | 42% | 24% | 11% |

**关键发现：**

1. **Easy 模式：** 最佳 agent 仅 52%——说明即使提供了完整方法，**一半的科研题目仍无法完成**
2. **Medium → Hard 断崖下降：** 从 31% → 15%，自规划和推理严重不足
3. **Hard 模式下最高 15%：** 没有一个 agent 能在真实科研自探索场景中超过 15%
4. **科学推理 ≈ 纯编码能力 + 领域知识：** 在跨领域迁移中，agent 表现极不稳定

## 🎯 核心洞见

### 1. "SWE-bench 不能衡量科研能力"

论文最有力的论点：**SWE-bench 本质上是代码修复/修改基准**。修复 bug → 运行测试是一种受限的、明确定义的任务。真实科研是不确定、开放式的、需要探索的——NatureBench 直接填补了这个评估空白。

### 2. 方法描述缺失 = 性能断崖

从 Easy 到 Medium（方法描述缺失）的 40% 性能下降表明：**当前 coding agent 缺乏补全不完备实验方法的能力**——这是科研代理的关键差距。

### 3. 没有 agent 超过 15%（Hard）

Hard 模式是真实科研的精髓——**只有题目，自规划自完成**。5 个顶级 agent 的最优是 15%，意味着**85% 的科研实验目前无法由 coding agent 自主完成**。

### 4. 领域知识的极端重要性

- 社会计算 → 58% Easy
- 量子化学 → 10% Hard
- 差距：5.8x

差距扩大提示领域知识的缺失**在困难任务中被放大**。

## ⚠️ 局限

1. **只有 52 题**：规模较小，统计效力需增大
2. **题目来自 Nature，但简化为编码任务**：真实科研中的资源获取、数据处理、failures的调试等维度未全覆盖
3. **自动化验证有限**：程序化验证器无法检查"科学发现是否真的有趣"
4. **时间限定**：每轮任务只给 30 分钟，部分题目可能需要更长时间

---

## 💡 启示

1. **NatureBench 是 SWE-bench 的有力补充**：SWE-bench 衡量的"改代码"能力 ≠ "做科研"能力
2. **当前 agent 的科研天花板约 15%**：真实自主科研还需要大幅提升
3. **领域知识缺失是核心瓶颈**：通用 LLM 需要更好的科学知识注入
4. **自规划是下一个重要能力**：方法描述缺失时的推理能力应成为 agent 训练的重点
5. **52 题看起来少，但每题是多步骤实验**：比 SWE-bench 的任务更有深度
