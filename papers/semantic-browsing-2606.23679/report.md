## 1. 问题陈述

**核心问题**: 现代文本到图像模型在视觉保真度和提示词遵守方面表现出色，但这种严格遵守是以牺牲多样性为代价的。当从同一提示词生成多个图像时，样本趋向于坍缩到同一个视觉解释，差异仅体现在视觉上不显著的细节中。

具体而言，该论文识别出三个层次的挑战：

1. **多样性坍缩（Diversity Collapse）**: 即使提示词是欠指定的（underspecified），不同的生成尝试也倾向于收敛到相同的高层语义解释，仅改变随机种子导致重复布局和有限变化。
2. **无意义变异**: 现有增强多样性的方法产生的输出由偶然变化（incidental variations）驱动，而非有意义的设计选择。
3. **缺乏用户控制**: 先前的多样性方法不提供对生成变化性质的显式用户控制，差异由随机效应驱动而非明确的语义因素。

**论文主张**: 需要一个新的多样性任务变体——**受控语义多样性（Controlled Semantic Diversity）**——在生成的样本上强制执行结构，使用户能够通过有意义、可解释的变化来探索。

## 2. 背景与动机

### 2.1 技术背景

现代文本到图像模型（如FLUX、DALL-E 3、FIBO）的核心训练范式强调对高度详细描述（elaborated captions）的严格遵守。这一设计选择显著提升了可控性和提示词忠实度，但也使模型倾向于承诺提示词的单一实现。

### 2.2 用户需求转变

随着模型能力提升，用户期望已经扩展：不再满足于单一忠实渲染，用户希望探索多个合理的输出，特别是在他们期望的结果尚不明确时。

### 2.3 核心洞察

论文的关键洞察是：**最近文本到图像模型在详细描述上的训练，实际上将语义决策与像素生成解耦了。** 这意味着可以在文本层面直接诱导多样性，而非依赖于 T2I 模型内部的随机变化。

这一洞察带来范式转变：
- **传统方法**: 依赖 T2I 模型内部的随机变化实现多样性
- **本方法**: 在文本层面明确指定语义变化，利用强提示遵守模型的能力精准渲染这些变化

---

## 3. 核心贡献

1. **新任务定义**: 提出「受控语义多样性（Controlled Semantic Diversity）」任务，要求生成的多样性具有可理解、可解释的变化轴。
2. **Semantic Browsing 框架**: 一种实现受控多样性的方法，用户可浏览结构化的图像画廊，沿有意义的、可解释的变化轴进行系统遍历。
3. **多智能体工作流**: 四个专门化智能体（Context Analyst、Brainstormer、Decision Maker、Critic）协同工作，在结构化场景空间中构建层级解释树。
4. **范式转变**: 从随机/潜在空间多样性转向文本层面、受显式语义约束驱动的多样性。
5. **全面评估**: 在 MS-COCO 数据集上的定量实验、消融研究、用户研究和结构评估，证明了方法在多样性和可导航性上的优势，同时保持图像质量和提示词遵守。

## 4. 相关工作：文本到图像的多样性

论文将与文本到图像多样性相关的工作分为几个流派：

### 4.1 CFG 相关方法

| 方法 | 核心思路 | 局限性 |
|------|---------|--------|
| **CFG (Classifier-Free Guidance)** | 放大条件信号提高对齐度 | 抑制多样性 |
| **Autoguidance (Karras et al., 2024)** | 用弱模型替换无条件模型 | 需要训练独立弱模型，计算密集 |
| **Gu & Hou (2025); Yehezkel et al. (2025)** | Autoguidance 的轻量替代 | 实践中可靠性有限 |

### 4.2 条件调制方法

| 方法 | 核心思路 | 局限性 |
|------|---------|--------|
| **CADS (Sadat et al., 2023)** | 去噪过程中向文本嵌入注入噪声 | 显著降低提示词对齐度 |
| **Guidance Interval (Kynkäänniemi et al., 2024)** | 仅在特定步长区间应用 CFG | 同上 |

### 4.3 潜在空间方法

| 方法 | 核心思路 | 局限性 |
|------|---------|--------|
| **Particle Guidance (Corso et al., 2023)** | 潜在空间中的排斥力 | 缺乏语义粒度 |
| **MinorityPrompt (Um & Ye, 2025)** | 基于损失的潜在优化 | 缺乏语义粒度 |
| **SGI (Parmar et al., 2025)** | 大规模种子池 + 过滤 | 受限于基础模型的固有多样性 |
| **Contextual Repulsion (Dahary et al., 2026)** | 注意空间中的排斥 | 仍依赖随机多样性，无显式控制 |

### 4.4 提示层面方法

| 方法 | 核心思路 | 局限性 |
|------|---------|--------|
| **PAG (Yun et al., 2025)** | GFlowNet 多样化采样 | 依赖训练数据，缺乏全局视图 |
| **Ours (Semantic Browsing)** | 层级树 + 多智能体 | — |

### 4.5 图像恢复中的多样性

论文也提及了图像恢复领域的相关工作：Cohen et al. (2024) 的「有意义多样性」和 Nehme et al. (2024) 的「层级探索」，但本文使用层级结构来组织显式的语义替代方案用于生成。

## 6. 相关工作：多智能体系统

| 方法 | 核心思路 | 与本工作的区别 |
|------|---------|---------------|
| **Maestro (Wan et al., 2025)** | 多模态智能体作为评审者优化输出 | 致力于收敛到单一「最佳」版本 |
| **PromptSculptor (Xiang et al., 2025)** | 分解复杂查询为详细描述 | 同上 |
| **Proactive T2I Agents (Hahn et al., 2024)** | 信令图明确歧义 | 同上 |
| **Twin-Co (Wang et al., 2025)** | 智能体反馈循环消除不确定性 | 同上 |
| **Ours (Semantic Browsing)** | 多智能体驱动探索而非收敛 | 探索多种解释，构建层级树 |

**关键区别**: 现有智能体系统致力于最大化对一个最优结果的控制，忽略了提示词的多种解释方式。我们的工作逆向而行，使用多个智能体驱动探索，构造层级树以确保系统生成广泛的创意结果。

## 7. 方法总览

Semantic Browsing 将生成过程形式化为在结构化场景空间内构建层级解释树（hierarchical interpretative tree）。

**核心工作流**:
1. 用户提示词 → JSON 场景表示（VLM 初始化）
2. 多智能体工作流迭代生成语义变体
3. 每个节点是完整可渲染的场景描述
4. 叶子节点通过 T2I 模型渲染为图像

**框架特征**:
- 每个分支点对应单一语义维度上的替代实现
- 之前固定的约束被保留（继承性约束）
- 包含「保留（preserve）」分支以沿其他维度继续探索
- 自动扩展 + 交互式浏览

## 8. 形式化定义与符号系统

| 符号 | 定义 |
|------|------|
| $\mathcal{S}$ | 完全指定的场景解释空间，编码为结构化 JSON |
| $p$ | 用户输入提示词 |
| $s_0 \in \mathcal{S}$ | 初始场景解释（通过 VLM 从 $p$ 扩展得到） |
| $(V, E)$ | 有根树，$V \subset \mathcal{S}$ 为节点集 |
| $c$ | 语义约束（semantic constraint），边缘的原子单位 |
| $a$ | 语义方面（semantic aspect），例如 subject interactions、scene composition、style |
| $(s_1, s_2) \in E$ | 存在约束 $c$ 将 $s_1$ 转换为 $s_2$ |
| $R(s, c)$ | VLM 场景精化器，应用约束 $c$ 到场景 $s$ |
| $C_s = (c_1, \dots, c_n)$ | 从根 $s_0$ 到节点 $s$ 的约束轨迹 |
| $\{d_i\}$ | 可变细节集合（Context Analyst 识别） |
| $\{a_i\}$ | 候选语义方面集合（Brainstormer 合成） |
| $a^*$ | 选定要变化的语义维度（Decision Maker 选择） |
| $\{c_i\}$ | 针对 $a^*$ 的替代语义约束集合 |

## 9. 树的三个核心要求

对于任何具有子节点的节点 $s$，语义约束集必须满足三个相互依赖的要求：

### 9.1 Semantic Structuring（语义结构化）
- 同一父节点的所有子节点必须来自共享的语义方面 $a$
- 确保每层的分支沿单一、语义上有意义的维度探索变化
- **示例**: 根节点的子节点都基于「Interactions」变化，右分支的子节点都基于「Dominance」变化

### 9.2 Heterogeneity（异质性）
- 每个约束 $c$ 必须以独特的方式实现共同的方面 $a$
- 多样性的主要驱动力
- **示例**: 在「Interactions」方面下，一个分支实例化「Lively play」，另一个分支实例化「Co-existing」
- 强制模型在同一共享方面内探索不同概念方向

### 9.3 Plausibility（合理性）
- 每个约束 $c$ 必须与原始提示词 $p$ 及其分支中先前的约束逻辑一致
- 合理性作为异质性的过滤器
- **示例**: 如果已建立「Co-existing」，后续「Cat dominates」约束必须在不具攻击性的情况下实现

> 这三个要求同时平衡是一个非平凡推理任务，因此需要多智能体工作流。

## 10. 多智能体工作流概述

树是逐节点扩展的，而非一次性生成。每次扩展节点 $s$ 时，四个专门化智能体按阶段处理：

```
Context Analyst → Brainstormer → Decision Maker → Critic → JSON Refiner → 更新树
```

**工作流输入**: 当前节点 $s$、用户提示 $p$、约束轨迹 $C_s$
**工作流输出**: 节点 $s$ 的子节点集合

### 智能体角色总结

| 智能体 | 负责的要求 | 核心任务 |
|--------|-----------|---------|
| **Context Analyst** | Plausibility | 定义可修改的搜索空间，识别可变细节 |
| **Brainstormer** | Semantic Structuring | 将低层细节合成为高层语义方面，评估变化幅度 |
| **Decision Maker** | Heterogeneity | 选择最高潜力维度，实例化为替代约束 |
| **Critic** | Plausibility | 验证提案，识别矛盾，精炼为可执行指令 |

## 11. Context Analyst（上下文分析师）

**负责**: Plausibility
**核心任务**: 定义可修改的搜索空间，直接操作于完整场景 $s$

### 工作原理

Context Analyst 基于以下洞察运作：生成的场景 $s$ 是由以下组成的复合体：
- **显式规范（explicit specifications）**: 提示词 $p$ 或积累约束 $C_s$ 强制执行的
- **未约束细节（unconstrained details）**: VLM 填充的未指定部分

通过明确区分这两者，Context Analyst 隔离出可变细节集合 $\{d_i\}$——例如特定的颜色、纹理或对象子类型。

**示例**: 场景包含"a dog, a cat, and a parrot"——必须存在，但其生物品种（如 Doberman vs. Samoyed）是可变细节。

**关键约束**: 一旦某个品种被添加到约束集中，对应场景细节在整个子树中固定。

### 消融影响
移除 Context Analyst 后：
- VQAScore（提示词遵守）保持不变（0.90）
- 层级一致性从 0.87 降至 0.82

证明 Context Analyst 对维持上下文的连续性（Plausibility 要求）不可或缺。

---

## 12. Brainstormer（头脑风暴者）

**负责**: Semantic Structuring
**核心任务**: 为有意义的语义结构化奠定基础

### 工作原理

给定原始提示词 $p$、积累约束 $C_s$ 和可变细节 $\{d_i\}$，Brainstormer：
1. 应用归纳推理将多个低层细节合成为一个高层语义方面 $a_i$
2. 显式评估每个候选方面的变化幅度：
   - 对场景**叙事（narrative）**的影响程度
   - 对场景**布局（layout）**的影响程度
   - 对场景**风格（style）**的影响程度
3. 将每个维度分类为高/中/低影响

**示例**: 在图 4 左侧分支中，不单独变化每个动物品种，而是将它们组合在"Breeds"方面下，实现协调修改。

**关键约束**: 通过优先考虑高影响力维度，确保树沿显著的概念转变而非琐碎变化演化。

---

## 13. Decision Maker（决策者）

**负责**: Heterogeneity
**核心任务**: 多样性生成的主要引擎

### 工作原理

1. 评估 Brainstormer 建议的候选方面 $\{a_i\}$
2. 识别特定于提示词（prompt-dependent，见附录 D）的、提供最丰富变异潜力的维度
3. 严格在提供的搜索空间内操作
4. 选择单一高影响力维度 $a^*$
5. 将 $a^*$ 实例化为一组替代语义约束 $\{c_i\}$
6. 主动推理场景的语义边界，制定广泛不同的解释而非增量调整

**关键设计**: 智能体被设计为明确阐述其推理过程（chain-of-thought），然后再最终确定任何行动。

---

## 14. Critic（评审者）

**负责**: Plausibility（验证层）
**核心任务**: 最终安全保障

### 工作原理

1. 根据原始提示词 $p$ 和积累约束 $C_s$ 推理提案约束
2. 识别可能出现的矛盾或歧义
3. 验证提案忠实地实现预期概念
4. 保持与原始提示词和积累上下文的严格对齐
5. 精炼候选集为精确、可执行的指令

**与自我纠正策略的关系**: 与 Madaan et al. (2023) 的 Self-Refine 和 Du et al. (2023) 的多智能体辩论一致。

### 消融影响
移除 Critic 后：
- VQAScore 从 0.90 降至 0.87（提示词遵守下降）
- 层级一致性保持不变（0.87）

证明 Critic 对于捕获上游智能体遗漏的罕见违规至关重要，防止语义漂移。

## 15. 交互式浏览（Interactive Browsing）

系统设计天生支持交互式浏览。除自动扩展策略外，用户可手动选择任意感兴趣的节点触发进一步生成，沿期望路径继续探索。

**两种选项**（每个节点）：
1. **提交（绿色）**: 提交所选语义方面的新实现，继续精炼该解释
2. **保留（橙色）**: 保留当前实现，从同一状态探索其他语义方面

所有节点对应有效的中间状态，可进一步扩展。

## 16. 实验设定

### 实现细节
- **基础 T2I 框架**: FIBO (Gutflaish et al., 2025)
- **VLM 默认**: Gemini 2.5 Flash
- **智能体配置**: 预设响应模板，4K-8K tokens，温度 0.4/0.7，最多 3 次重试
- **树结构**: 分支因子 3，深度 3 → $3^3 = 27$ 个叶子节点

### 数据集
- 50 个提示词随机采样自 MS-COCO 数据集 (Lin et al., 2015)

### 评估指标

| 维度 | 指标 | 说明 |
|------|------|------|
| **多样性** | Vendi Score ↑ (Inception 空间) | 语义变化程度 |
| **多样性** | DINO Similarity ↓ (成对) | 图像间语义相似度 |
| **图像质量** | Aesthetic Score ↑ (LAION 预测器) | 视觉美学质量 |
| **提示遵守** | VQAScore ↑ | 提示词-图像对齐 |

### 基线方法

| 基线 | 类型 | 说明 |
|------|------|------|
| **Stochastic VLM Seeding** | VLM 随机性 | 变化 VLM 随机种子生成 27 张 |
| **Post-Hoc Diversity Opt.** | 生成-选择 | 79 候选→QIP 选择 27，匹配 LLM 调用预算 |
| **High-Temp. Post-Hoc Opt.** | 生成-选择 + 高温 | 同上，增加采样温度 |
| **CADS** | 去噪过程 | 向文本嵌入注入随机噪声 |
| **Guidance Interval** | CFG 调制 | 仅中间步长应用 CFG |
| **Power-Law CFG** | CFG 缩放 | CFG 更新乘以范数的幂 |

所有基线使用相同的底层模型（FIBO），超参数针对多样性优化。

## 17. 定性评估

### 主要发现
1. **结构化多样性**: 输出组织为三元组，每组共享唯一祖先节点，展示早期分支决策如何传播为不同但内部一致的变体。

## 20. 结构评估：语义-拓扑相关性

### 假设
两张图像之间的语义距离应与其在生成树中的拓扑距离相关。这是 Semantic Structuring 要求的直接结果——父子节点之间恰好一个方面变化。

### 方法
- 分析成对 DINO Distance 作为图距离（节点间路径长度）的函数
- 50 棵树 × 所有节点对 = 17,550 个数据对

### 结果（图 11）

| 图距离（hops） | 中位 DINO Distance |
|----------------|-------------------|
| 1 hop | 0.168 |
| 2 hops | — |
| 3 hops | — |
| 4 hops | — |
| 5 hops | 0.452 |

中位 DINO Distance 从 1 hop 的 0.168 单调上升到 5 hops 的 0.452，证实：
- 相邻节点共享视觉特征
- 远处节点展现更大的语义分歧
- 层级结构有效编码语义关系

## 21. 结构评估：层级一致性

### 方法
- 使用 LLM-as-a-judge (Lee et al., 2024)
- 测量生成节点与其祖先继承约束之间的对齐度
- 显式验证 Plausibility 要求（Section 3.2）

### 结果
- 层级一致性得分：**0.87/1.0**
- 在绝大多数情况下，生成节点成功遵守根到节点的累积约束

### 评估细节
- 仅惩罚约束的首次违反
- 子节点若与已违反约束的父节点一致则不累积惩罚
- 该方法可精确定位违规发生的位置

## 26. 局限性与未来方向

### 局限性

1. **生成模型依赖**: 探索的语义空间质量和有用性依赖于底层生成模型忠实实现细粒度提示修改的能力，以及智能体的语义推理能力。
2. **VLM 创意广度有限**: 现代 VLM 在维持一致性和合理性方面有效，但提出丰富多样语义替代方案的能力相对有限，可能约束语义空间的广度。
3. **约束累积效应**: 更深的树中提示遵守逐渐下降（VQAScore 从 0.81 降至 0.75），提示约束积累带来的负担。

### 未来方向

1. **跨模态拓展**: 结构化语义探索可扩展到图像以外的领域——视频、3D 内容、多模态生成。
2. **原则性范式**: 从孤立输出到连贯、可导航的替代空间的转变。
3. **更轻量的推理**: 在当前任务制定中，两个"保留"分支在深度方向的每个节点与两个"变化"分支一起被持久化。在 27 张图像的画廊中，叶子节点包含 2³ = 8 张独特的变化图像和 19 张身份映射图像。寻找更高效的结构表达是未来方向。
4. **更强的 VLM 骨干**: 随着 VLM 的创意广度增加，Semantic Browsing 的语义空间广度也将自然增长。

## 27. 总结与启示

### 高层视角

这篇论文采取的视角是：**多样性源于显式语义决策，而非仅来自随机变化。** 通过承诺具体的语义选择，输出间的差异变得可解释且持久，而非偶然。

### 范式转变

论文的核心理念是：**将强提示遵守视为多样性的赋能者而非限制。** 最近 T2I 模型对详细描述的严格遵守实现了精确的语义控制，使得在文本层面构建结构化多样性成为可能。

### 关键设计选择

1. **结构化 JSON** 作为场景表示——提供细粒度控制
2. **层级树**——提供结构化的变化组织
3. **多智能体**——并行满足 Semantic Structuring、Heterogeneity、Plausibility 三个需求
4. **继承性约束**——之前固定的决策保持一致性

### 对社区的启示

1. 多样性评估不仅需要考虑数量，还需考虑可导航性和可解释性
2. 语义-拓扑相关性可作为结构化多样性评估的新范式
3. 多智能体系统可从"收敛到最佳"转向"探索可能性空间"
4. 强提示遵守模型是结构化多样性的理想基础
