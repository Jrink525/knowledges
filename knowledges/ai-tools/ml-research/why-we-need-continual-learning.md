# 我们为什么需要持续学习 —— 从 Memento 困境到参数级记忆

> **作者：** Malika Aubakirova & Matt Bornstein，a16z AI Infrastructure 团队  
> **发布于：** 2026 年 4 月 22 日  
> **核心理念：** 部署后的模型应该能持续压缩新经验到参数中，而不只是依赖外部检索。这是一个跨越架构、安全、创业生态的前沿研究方向。

> 本文汇集了大量研究者、博士生和创业创始人的第一手洞见，是国内少见的系统性综述。

---

## 引言：Memento 困境

> *"在克里斯托弗·诺兰的《记忆碎片》中，Leonard Shelby 活在破碎的当下……他靠纹身和宝丽来照片作为外部道具来提醒自己，他的大脑已经无法保留任何新信息。大语言模型活在类似的永恒当下中。"*

LLM 的训练过程将互联网压缩进了参数，但部署之后——**参数冻结，记忆终止**。

| | 人类（正常） | LLM（当前） |
|--|-----------|-----------|
| **新经验处理** | 内化为长期记忆 | 仅靠上下文"便签" |
| **学习机制** | 突触可塑性（持续压缩） | 权重冻结，外部检索替代 |
| **失败模式** | 遗忘（渐进） | 上下文溢出（突然崩溃） |
| **长期成长** | 复利效应 | 零增长 |

当前的做法是把模型用"脚手架"围起来：聊天历史如即时贴，RAG 如外部笔记本，系统提示词如纹身。但模型本身从未真正内化新信息。

> **ICL 是暂时的。真正的学习需要压缩。** —— a16z

---

## 第一章：先承认——上下文真的有用

在论述参数级学习之前，作者先承认 ICL 行之有效，且有坚实理由相信它会继续赢。

### "丑陋但原生"的优势

Transformer 本质上是条件性下一个词预测器。给它正确的序列，不需要动权重就能获得丰富行为。Prompt engineering、 instruction tuning 和 few-shot 都因此强大。

**两个例证：**

- **Cursor（Windsurf）** 的自主编码 Agent 扩展报告中写道："系统的行为令人惊讶地大部分来自我们如何提示 Agent。Harness 和模型很重要，但提示更重要。"——权重固定，变的是上下文编排。
- **OpenClaw** 的成功不是因为特殊的模型访问（底层模型谁都能用），而是因为它有效地将上下文和工具整合为工作状态：追踪你在做什么、结构化中间产物、决定什么重新注入提示、维护持久记忆。OpenClaw 将 Agent Harness 设计提升为一门学科。

> "丑陋但原生"的接口往往能赢，因为它们与底层系统耦合，而非对抗它。

### SSM：增强版上下文

从 LLM 调用到 Agent Loop 的演进，让上下文压力剧增。Agent 在执行 **20–100 步后经常失败**——上下文填满、连贯性退化、停止收敛。

主流方案：**状态空间模型（State Space Models, SSMs）**——在正常注意力头之间穿插固定记忆层。目标是将 Agent 从 ~20 步维持到 **~20,000 步**。

| | 传统 Transformer | SSM |
|--|-----------------|-----|
| **长上下文缩放** | O(n²) | O(n) |
| **Agent 步数** | ~20–100 步 | ~20,000 步 |
| **记忆层** | 一次性上下文 | 可持久化外部记忆 |

SSM 虽不更新权重，但引入了几乎不需重置的外部记忆层——可以视为一种**准持续学习**。

## 第二章：上下文错过了什么——"文件柜谬误"

> *"如果一个 AGI 系统有无限存储空间，世界上最大的文件柜，每个事实都被完美索引、瞬间可检索……它学到了吗？没有。它从未被迫进行压缩。"*

### Ilya Sutskever 的核心论点

> *"预训练在某种意义上已经超出了目标……人类不是 AGI。是的，人类有技能基础，但人类缺乏大量的知识。相反，我们依赖持续学习。如果我培养出一个超级聪明的 15 岁少年，他们知道得并不多。一个很棒的学生，非常渴望学习。你可以说'去当程序员，去当医生'。部署本身涉及某种学习、试错的过程。这是一个过程，不是投放成品。"*

LLM 本质上是**压缩算法**。在训练中，它们将互联网压缩进参数。这个压缩是有损的——而这恰恰是它的力量所在。压缩迫使模型找到结构、泛化、建立跨上下文迁移的表征。在部署时停止压缩 = 停止学习。

### 费马大定理的启示

Yu Sun（TTT 作者）用一个数学例子说明了这个辩论：费马大定理 350 多年无人能证，不是因为缺乏相关文献，而是因为解法高度新颖。怀尔斯在 1990 年代花了七年时间，独自发明了新技术——将椭圆曲线和模形式两个分支结合在一起。

**核心问题：** 这证明了 LLM 缺少某种更新先验的能力？还是证明只要规模够大，LLM 就能做到一切？答案是**实证性的，目前未知**。

### 语言无法传递的知识

有些模式维度太高、太隐晦、太深结构，无法塞进上下文：

| 知识类型 | 语言可描述程度 | 只能存在于 |
|---------|-------------|----------|
| 医学影像中良性与恶性的纹理差异 | 约≈60% | 权重（latent space） |
| 说话者独特韵律的微波动 | 约≈50% | 权重 |
| 某人的思维方式模式 | 约≈30% | 权重 |
| 具体事实（"法国首都是巴黎"） | ≈100% | 上下文即可 |

> 这解释了为什么 ChatGPT 的"机器人记住你"功能经常让用户感到不适，而非愉悦。用户要的不是**回忆**，而是**胜任力**。一个模式化的模型能泛化到新场景；一个只回忆历史的不能。

## 第三章：持续学习入门——关键在于压缩发生在哪里

### 三板斧：Context → Modules → Weights

**不是"有/无记忆功能"之分，而是"压缩发生在哪里"之分。**

```
零压缩（纯检索，权重冻结）    中间态（模块）         全内化（权重级学习）
    |                            |                         |
  上下文增强                   可插拔模块               参数更新
  (RAG, Agent Harness)        (KV cache, adapter)       (TTT, RL, meta-learning)
```

### 3.1 上下文端（最成熟）

- 更智能的检索管线、Agent Harness、提示编排
- 基础设施成熟，部署干净
- **局限：** 上下文的深度——即使 SSM 扩展到 20K 步，仍是上下文外挂

**值得注意的新方向：多 Agent 架构作为上下文的缩放策略**

> "如果一个单模型被 128K 上下文窗口限制，一个协调的 Agent 群体——每个持有自己的上下文、专门处理问题的一个切片、通信结果——可以集体逼近无界工作记忆。"

Karpathy 的 auto-research 项目和 Cursor 构建浏览器的例子都是早期验证。

### 3.2 模块端（中间地带）

- 可插拔知识模块：压缩的 KV 缓存、Adapter 层、外部记忆存储
- **关键承诺：** 8B 模型 + 合适模块可以在特定任务上匹敌 109B 模型，内存仅一小部分
- 兼容现有的 Transformer 基础设施

### 3.3 权重端（最深度，最难部署）

这才是真正的参数级学习：

| 研究方向 | 核心机制 | 代表工作 |
|---------|---------|---------|
| **正则化 & 权重空间** | 按重要性惩罚参数变化 | EWC (Kirkpatrick 2017), 权重插值 (Kozal 2024) |
| **Test-Time Training (TTT)** | 推理时对新数据执行梯度下降 | Sun et al. (2020), TTT 层, TTT-E2E, **TTT-Discover** |
| **元学习 (Meta-learning)** | 训练模型学会如何学习 | MAML (Finn 2017), **HOPE / Nested Learning** (Behrouz 2025) |
| **蒸馏 (Distillation)** | 学生匹配冻结教师输出 | LoRD (Liu 2025), **SDFT** (Shenfeld 2026) |
| **递归自改进** | 从自生成数据中提升 | STaR (Zelikman 2022), AlphaEvolve (DeepMind 2025) |

> **这些方向正在融合。** TTT-Discover 已融合 TTT + RL 探索。HOPE 在单一架构中嵌套快/慢学习循环。SDFT 将蒸馏转化为自改进原语。下一代持续学习系统将组合多种策略：用正则化稳定、元学习加速、自改进复利。

## 第四章：持续学习 Startup 全景

### 非参数端（最熟悉）

| 子类别 | 代表公司 | 核心能力 |
|-------|---------|---------|
| **Harness（编排层）** | Letta, mem0, Subconscious | 管理什么进上下文窗口 |
| **外部存储 & RAG** | Pinecone, xmemory | 提供检索骨干 |

> 数据已经存在，挑战是**在正确的时间将正确的数据切片放到模型面前**。

### 参数端（更早期，更多样）

| 方向 | 核心赌注 | 关键论点 |
|------|---------|---------|
| **局部压缩**（不重训） | 可插拔知识模块（KV cache / adapter） | 隔离学习的 plasticity-stability 更易管理；8B + 模块 ≈ 109B |
| **RL & 反馈循环** | 把每一次交互视为训练信号 | 部署循环中已有最丰富的信号（用户纠正、任务成功/失败） |
| **数据驱动** | 瓶颈在学习算法，在数据本身 | 高质量的精选/合成数据 = 更少的梯度步 |
| **全新架构** | Transformer 本身就是瓶颈 | 需要连续时间动力学 + 内建模记忆的原语 |

> 所有主要 AI 实验室都在各方向活跃。领域足够早期，没有单一方法胜出——考虑到用例的多样性，**也不该有**。

## 第五章：为什么朴素权重更新会失败

### 工程问题（已充分记录）

| 失败模式 | 现象 | 原因 |
|---------|------|------|
| **灾难性遗忘** | 学到新数据时摧毁已有表征 | 稳定性-可塑性困境 |
| **时间纠缠** | 不变规则与可变状态被压缩进相同权重 | 更新一个会破坏另一个 |
| **逻辑整合失败** | 事实更新不传播到其逻辑后果 | 权重更新局部于 token 序列，非语义概念 |
| **无法卸载** | 虚假/有毒知识无法手术式移除 | 没有"减法"可微算子 |

### 安全问题（讨论更少的深层问题）

训练与部署的分离不仅是工程便利——它还是**安全、审计和治理的边界**。打破它，多方同步崩溃：

| 问题 | 后果 |
|------|------|
| **对齐退化** | 即使在良性数据上做窄微调，也可能产生广泛的不对齐行为 |
| **数据投毒面** | 持续更新创造了一个缓慢性、持久版本的 Prompt Injection——活在权重里 |
| **审计断裂** | 持续更新的模型是移动靶标——无法版本化、回归测试、或一次性认证 |
| **隐私风险** | 用户交互被压缩进参数——信息存留在远比检索上下文更难过滤的表征中 |

> 这些都是**开放问题，不是根本不可能**，解决它们与解决核心架构挑战一样是持续学习研究方向的一部分。

## 终章：从 Memento 到记忆

> *"Leonard 的悲剧不在于他不能运作——在任何给定场景中他都有资源、甚至很聪明。他的悲剧在于他永远无法复利。每一次经验始终是外部的——一张宝丽来、一个纹身、别人笔迹的笔记。他可以检索，但他无法压缩新知识。"*

**作者对未来的预测：** 前进之路不是一个单一的突破，而是一个**分层系统**：

```
第一层：ICL —— 原生、经过验证、持续改进（上下文增强）
第二层：Modules —— 个性化和领域专精的中间地带（可插拔知识模块）
第三层：Weights —— 真正的参数级压缩（TTT、元学习、自改进循环）
```

> "文件柜越来越大了。但更大的文件柜仍然是文件柜。真正的突破是让模型在部署之后，做那些在训练中让它变得强大的事：**压缩、抽象、学习**。"

## 知识库关联

- **[Agent Complexity Ratchet](../agent-engineering/agent-complexity-ratchet.md)** — Garry Tan 的"复利"论点是本文"从 Memento 到记忆"这一论点的工程实践版：个人 Agent 系统的复利根植于上下文编排，而本文论证这还不够，需要参数级学习。
- **[Agent Harness Engineering](../agent-engineering/agent-harness-engineering.md)** — Addy Osmani 的 Harness 正是本文"非参数端"的实践详解。本文说"Harness 工作得很好，但天花板存在"，OS 文是"Harness 如何工作到最好"。
- **[4-Agent Content Production System](../agent-engineering/4-agent-content-production-system.md)** — 本文提到"多 Agent 架构作为上下文缩放策略"——4-Agent 系统正是放大上下文容量而不触及权重的具体实现。
- **[10 Claude Code Agents Pipeline](../agent-engineering/10-claude-code-agents-pipeline.md)** — @zodchiii 的 10 Agent 管线是本文"协调的 Agent 群体"概念的实战验证。
- **[Claude 五个能力等级框架](../claude/claude-five-levels-framework.md)** — Nate Herk 的 L5 "Cloud Routines"本质上需要一个持续学习能力——Agent 今天能做 L1-L4 的事，L5 要求模型从部署经验中学习。

### 核心洞见速查

| 洞见 | 行动 |
|------|------|
| "文件柜谬误"——检索≠学习 | 不要被"无限上下文"迷惑，压缩才是学习 |
| SSM 将 Agent 步数从 20→20K | 评估 Agent 框架时关注其记忆架构，不只是上下文长度 |
| TTT 层让推理时梯度下降成为可能 | 这是最接近"部署后学习"的可行路径 |
| 安全/审计/隐私是比遗忘更难的问题 | 持续学习的工程实现必须从第一天考虑治理边界 |
| OpenClaw 是"上下文编排获胜"的例子 | Harness 设计是一门学科——足以支撑当前的整个 AI 应用生态 |
