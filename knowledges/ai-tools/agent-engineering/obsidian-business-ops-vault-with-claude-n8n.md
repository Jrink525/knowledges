# 搭建一个在你睡觉时自动运转业务的 Obsidian Vault（完整教程）

> **来源：** [How to Build an Obsidian Vault That Runs Your Entire Business While You Sleep](https://x.com/cyrilxbt/status/2054379666316693719)
> **作者：** cyrilXBT (@cyrilxbt)
> **日期：** 2026-05-12

---

## 核心洞见

大多数生意人每天做两种工作：**推动业务前进的工作**（战略、关系、创造性判断）和**防止业务倒退的工作**（行政杂务、状态更新、报告、信息处理）。

后一种正在吃掉前一种。

> 普通知识工作者每天花 4-6 小时在行政协调上。这**不是效率问题，是架构问题。**

解法不是让你更快地做行政工作，而是**把你从行政工作中完全移除**。

---

## 一、先理解架构，再谈工具

一个业务操作系统不是笔记的集合。它是一个**互联的系统网络**，信息自动从产生的地方流向需要的地方，动作无需你手动触发。

三层架构：

```
┌────────────────────────────────┐
│  Automation Layer: N8N          │ ← 定时器、触发器、神经网
│    调度、路由、通知              │
├────────────────────────────────┤
│  Intelligence Layer: Claude MCP │ ← 处理、思考、生成
│    读文件、做连接、推输出         │
├────────────────────────────────┤
│  Knowledge Layer: Obsidian      │ ← 纯文本 Markdown 文件
│    永不锁定、永不失效             │
└────────────────────────────────┘
```

- **Knowledge Layer** = Obsidian + 纯文本 Markdown。每一条业务信息都存在这里，格式可读、可搜索、永不锁定。
- **Intelligence Layer** = Claude Code 通过 MCP 协议连接。把你的 Vault 从被动仓库变成主动操作系统。
- **Automation Layer** = N8N。定时触发和事件驱动的工作流引擎。让系统在无人状态下持续运转。

## 二、Vault 结构设计：面向业务流程，而非面向主题

个人知识管理型 Vault 按主题组织文件夹；业务操作系统型 Vault **按业务流程组织**。

```
01 - CLIENTS/                    ← 所有客户档案
   [Client Name]/
       overview.md               ← 客户概况
       communications/           ← 沟通记录
       contracts/                ← 合同
       deliverables/             ← 交付物

02 - PROJECTS/                   ← 所有项目
   [Project Name]/
       overview.md               ← 项目概况
       tasks.md                  ← 任务列表
       deliverables/             ← 交付物

03 - OPERATIONS/                 ← 运营标准
       content/
           calendar.md           ← 内容日历
           performance.md       ← 数据追踪
       finances/
           revenue.md
           expenses/
       processes/                ← SOP

04 - DAILY/                      ← 每日记录
       [YYYY-MM-DD].md

05 - CLIENTS/ (备选)              ← 潜在客户

06 - SYSTEM/                     ← 系统核心
       CLAUDE.md                 ← 业务宪法（最重要的文件）
       vault-config.md

07 - QUEUE/                      ← 异步任务队列
       DRAFT-[type]-[topic].md
       RESEARCH-[topic].md

08 - GENERATED/                  ← 系统产出
       briefings/
       drafts/
       reports/
       communications/
```

**关键机制：`QUEUE` 和 `GENERATED` 是系统的运行核心。**

- `QUEUE` = 你和智能层异步沟通的接口。放下需求文件，系统下次运行时处理。
- `GENERATED` = 系统产出存放处。你随时查看结果。

> **你想到任务就记录，系统按周期执行，你得空时 review 结果。** 这个异步循环就是"让你睡觉时业务还在运转"的秘密。

## 三、CLAUDE.md：你的业务宪法

`SYSTEM/CLAUDE.md` 是整个业务操作系统里**最重要的单个文件**。每个自动化工作流在执行任务前都会先读它。

**一个写得好 vs 写得差的 CLAUDE.md，区别在于系统输出是"需要大改"还是"几乎可用"。**

CLAUDE.md 核心结构：

```
# Business Identity
- 你是谁、你做什么、你的价值主张
- 目标客户画像
- 品牌调性（语气风格指南）

# Business Context
- 当前业务阶段（growth / scale / pivot）
- 主要收入来源
- 核心团队和角色

# Weekly Focus (每周一更新)
- 本周最重要的 3 件事
- 本周需要关注的风险

# Content Pillars
- 3-5 个内容支柱，每个附带：
  - 目标受众
  - 典型的选题方向
  - 成功标准

# Voice Profile
- 写作风格说明
- 习惯用语和禁忌词
- 示例段落

# Standard Operating Procedures
- 各系统的工作流说明
- 输出格式要求
- 审核标准
```

> **每周一更新 Weekly Focus 部分**——2 分钟的事，但能让接下来一周所有自动输出的相关性大幅提升。

## 四、六个业务系统详解

### 系统 1：客户情报系统

**确保你对每个客户关系都有完整上下文，无需手动维护。**

**1a. 会前简报（Pre-Call Brief）**
- 触发：N8N cron，会议前 30 分钟
- 动作：读取客户文件夹 + 最近沟通记录 → 生成 5 分钟可读完的会前简报
- 产出：涵盖关系时间线、当前状态、待办项、上次沟通、建议议程、最重要的一句话
- 保存至：`08 - GENERATED/briefings/[日期]-[客户]-precall.md`

**1b. 沟通日志（Communication Logger）**
- 你在每日笔记中记录任何客户互动
- 系统自动提取关键信息，格式化后追加到客户文件夹
- **记录一次，永久归档到正确位置。**

### 系统 2：项目运营系统

**保持每个活跃项目自动推进，无需手动追踪状态。**

**2a. 每日项目脉搏（Daily Project Pulse）**
- 触发：每天早上 6AM cron
- 动作：扫描所有活跃项目文件夹 → 生成状态摘要
- 自动标记 7 天内到期未完成、连续 5 天状态不变、被阻塞无解决方案的项目
- 产出：`08 - GENERATED/briefings/[日期]-project-pulse.md`

**2b. 项目自动更新器（Project Auto-Updater）**
- 你在每日笔记中用 `DONE: [项目名] — [交付物]` 格式记录完成
- 系统自动更新对应项目的任务列表
- **你干活，项目文件自己更新。**

### 系统 3：内容生产系统

**你不再是内容产出的瓶颈。**

**3a. 每日内容简报（Morning Content Brief）**
- 读取：过去 48h 的研究简报 + 你的每日笔记灵感 + 内容日历 + CLAUDE.md 内容支柱
- 产出：3 个内容角度排序 + 各角度的钩子方案 + 推荐格式 + 是否需要额外研究
- 保存至：`08 - GENERATED/briefings/[日期]-content-brief.md`

**3b. 草稿引擎（Draft Engine）**
- 在 `QUEUE` 中放入 `DRAFT-[类型]-[主题].md` 文件
- 队列处理器读取你的研究简报，应用 CLAUDE.md 中的语气档案，生成初稿
- **80% 的内容只需要不到 10 分钟的编辑即可发布。**

**3c. 内容表现追踪（Performance Tracker）**
- 每周一自动拉取全平台上周数据
- 输出：Top 3 最高表现内容 + 最低表现内容 + 表现差异分析 + 下周优化方向

### 系统 4：财务运营系统

**所有数字自动更新，无需单独的记账流程。**

**4a. 收入记录**
- 每日笔记中用 `RECEIVED: $[金额] — [客户] — [描述]` 记录
- 系统自动：更新月度收入追踪 → 追加到客户发票文件夹

**4b. 月度财务简报**
- 每月 1 号自动生成
- 包含：总收入 vs 目标、环比、按客户分类收入、Top 费用类别、净收入、下月趋势预测

**4c. 发票提醒**
- 每月 25 号自动检查 retainer 客户付款记录
- 生成草稿发票邮件到 `GENERATED/communications`，你审核后发送
- **你审批，系统起草。**

### 系统 5：研究与情报系统

**无需花费数小时阅读，系统帮你消化。**

**5a. 每日情报简报**
- 每天早上自动拉取：24 小时内的 AI/行业新闻、竞争对手动态、品牌提及
- 产出：3 个最重要的行业动态 + 一个商业机会 + 一个商业风险 + 建议行动

**5b. 研究队列**
- 在 `QUEUE` 中放入 `RESEARCH-[主题].md` 文件
- 研究工作流：读请求 → 搜索相关资料 → 交叉验证 vault 已有知识 → 输出结构化研究简报
- **你半夜提交研究请求。6AM 简报已就绪。**

### 系统 6：绩效与回顾系统

**闭环检验所有其他系统的产出。**

**6a. 每周回顾**
- 触发：每周日 8PM
- 读取：7 天每日笔记 + 项目状态变更 + 内容表现报告 + 财务记录 + 本周所有产出
- 产出：推动的是什么/没推动的是什么/本周出现的模式/下周 3 个优先级/本周对自身模式的 insight

**6b. 季度业务回顾**
- 读取三个月每周回顾 → 生成战略复盘
- 曾经需要一整天的分析工作，现在只需要 20 分钟阅读。

## 五、N8N 自动化层设置

**硬件：** 自托管 N8N 在 $5/月的 DigitalOcean droplet 上 —— 不限工作流次数，无按次计费。

**工作流触发器类型：**

| 类型 | 用途 | 示例 |
|---|---|---|
| Cron Trigger | 定时操作 | 早 6AM 简报、周日 8PM 周报、每月 1 号财务简报 |
| File Watch | 事件驱动 | QUEUE 文件夹新文件、项目文件夹变更 |
| Webhook | 外部事件 | Stripe 付款触发收入更新、新日历事件触发会前简报 |

**核心模式（每个 Claude 驱动的工作流都遵循）：**

```
Trigger → 读 CLAUDE.md → 读相关 Vault 内容
   → 调用 Claude API（附上编译后的上下文）
   → 写回 Vault → 发送通知 → 记录操作日志
```

**关键：** Claude API 调用始终将 CLAUDE.md 内容作为 system context。这是让每个自动输出**业务特定化**而非泛化的关键。

## 七、复合效应

最重要的不是第一周系统做了什么，**而是第六个月系统能做什么。**

- 每一次记录的客户互动，都让会前简报更丰富。
- 每一个内容表现数据点，都让内容简报更准确。
- 每一份研究简报，都增加了系统可调用的知识深度。
- 每一次周报，都成为季度分析的素材。

> **系统在复合增长。** 持续使用六个月后，你的 Vault 对业务的了解深度，需要一个人类员工花几年才能达到。

**先搭一个最小的启动组合：CLAUDE.md + 每日简报工作流。**

然后每周加一个系统。两个月后，你有一套完整运转的业务操作系统。六个月后，你无法想象没有它怎么经营。
