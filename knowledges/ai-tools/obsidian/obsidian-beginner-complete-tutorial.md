## 一、为什么需要本地知识库

AI 本身没有立场、没有经历、没有审美。直接问它问题，回答正确但没有灵魂；让它代写文章、做方案、整理录音——它给的永远是通用、平均、谁都能写出来的内容。

**差别在哪？** 在于你能不能把"你自己的素材、方法论、经历、判断"喂给它。同样的模型，喂进去你的东西，输出的就是带你自己印记的内容。

这就是为什么人人都该拥有一座**本地知识库**——不只是程序员，不只是知识工作者。

而本地知识库的最佳方案，就是 **Obsidian**。

## 二、Obsidian 是什么

一句话：**本地的、基于 Markdown 文件的笔记软件**，最大特色是双向链接和插件生态。

### 与普通笔记软件的三个本质区别

| | 普通笔记（Notion/印象等） | Obsidian |
|---|---|---|
| 数据存储 | 云端/私有格式 | **本地 .md 文件** |
| 关联方式 | 文件夹分类 | **双向链接**织成知识网络 |
| 可扩展性 | 平台限制 | **插件生态**无限扩展 |

### 收费 & 适用人群

- **收费吗？** 个人使用完全免费。只有"云同步"和"在线发布"两个付费服务，新手用不到
- **适合谁？** 笔记体量大、希望长期沉淀知识的人。偶尔记购物清单的话，备忘录就够

## 三、安装与初始设置

### 下载

访问 [obsidian.md/download](https://obsidian.md/download) 下载对应版本。

- **Windows：** 下载 .exe → 双击安装 → 一路下一步 → 桌面出现紫色图标
- **macOS：** 下载 .dmg → 双击打开 → 拖到 Applications 文件夹 → 启动台打开
- **iOS / Android：** App Store / Play Store 搜「Obsidian」安装。建议先在电脑上跑通再装手机版

### 创建第一个仓库（Vault）

启动后 → 「创建新仓库」→ 取名字（如"我的笔记"）→ 选位置（桌面或文档文件夹，别放 C 盘深处）→ 创建

### 改成中文界面

设置（左下齿轮）→ About → Language 选「简体中文」→ Relaunch 重启

### 界面五大区

1. **文件列表**（左侧）— 仓库内所有笔记
2. **编辑区**（中间）— 写笔记的地方
3. **右侧栏** — 内置/插件的辅助面板（日历、图谱、大纲等）
4. **状态栏**（底部）— 字数、插件状态等
5. **功能区**（左上/左中）— 搜索、切换、命令面板等

## 四、核心概念

### 4.1 仓库（Vault）

一个文件夹，里面存着所有笔记。新手先用一个仓库就够，别一上来建好几个。

### 4.2 Markdown

纯文字标记格式。不需要 Word 的点工具栏——在文字前打 `# 加空格` 就是标题。语法很少，用着用着自然就会。

**速查表：**

| 效果 | 写法 |
|---|---|
| 一级标题 | `# 标题` |
| 二级标题 | `## 标题` |
| 三级标题 | `### 标题` |
| **加粗** | `**文字**` 或 `Ctrl/Cmd + B` |
| *斜体* | `*文字*` 或 `Ctrl/Cmd + I` |
| - 列表 | `- 项目` |
| 1. 有序列表 | `1. 项目` |
| 代码 | `` `行内代码` `` |
| 代码块 | ```` ```语言 ```` |
| 链接 | `[文字](url)` |
| 图片 | `![描述](路径)` |
| 引用 | `> 引用文字` |
| 分割线 | `---` |
| 表格 | `\|列1\|列2\|` |

### 4.3 双向链接（核心）

在 A 笔记里写 `[[B笔记]]`，两件事发生：
1. A 笔记里出现了一个可点击的链接，跳到 B
2. B 笔记的**反向链接**面板会自动显示：有笔记提到了我

这就是"双向"。笔记越多，这张网越值钱。

**新手记一件事：** 在笔记里用 `[[词]]` 把词括起来就创建了链接。如果这篇笔记不存在，点击它会自动新建。

### 4.4 插件

**第一天不要碰社区插件。** 先把核心插件里基础的用起来，等用了一两周发现某个原生满足不了需求再去搜。

### 4.5 三种编辑模式

- **编辑模式** — 看到 Markdown 源码，直接修改
- **阅读模式** — 渲染后的美观显示，不能编辑
- **实时预览** — **最推荐**。所见即所得，写的时候就看到渲染效果

**快捷键：** `Ctrl/Cmd + E` 切换编辑/阅读模式

## 五、配置建议

### 必调设置

- **设置 → 文件与链接 → 自动更新内部链接：** 开（改文件名时自动更新所有引用）
- **设置 → 外观 → 主题：** 官方推荐 Blue Topaz（易用好看），或用本来的默认主题
- **设置 → 核心插件：** 启用「日记」「模板」「大纲」「文件恢复」
- **设置 → 文件与链接 → 附件默认存放路径：** 设一个文件夹如 `附件`，避免附图散落各处

### 高频快捷键

| 操作 | 快捷键 |
|---|---|
| 新建笔记 | `Ctrl/Cmd + N` |
| 打开命令面板 | `Ctrl/Cmd + P` |
| 快速切换笔记 | `Ctrl/Cmd + O` |
| 搜索 | `Ctrl/Cmd + Shift + F` |
| 切换编辑/阅读 | `Ctrl/Cmd + E` |
| 插入双向链接 | `[[` |

更多快捷键：设置 → 快捷键 → 查看和自定义

### 两个必记肌肉记忆

1. `Ctrl/Cmd + O` 敲回车 → 新建笔记并打开（最快的新建方式）
2. `Ctrl/Cmd + Shift + F` → 全文搜索仓库里的任何内容

## 六、组织方法论

**核心原则：** 文件夹保持浅层（不超过 1-2 级），靠链接和标签做主组织。**没写过几篇笔记就开始设计分类，必翻车。**

### MOC（内容地图）

建一篇"目录笔记"，主要不是写正文，而是把同主题的其他笔记按结构列出来：

```markdown
# 数据库学习 · MOC

- [[数据库基础概念]]
  - 事务 ACID
  - 索引类型
- [[SQL 高级查询]]
  - JOIN
  - 窗口函数
```

可建一篇 Home 笔记列出所有 MOC，形成"地图的地图"。这是最受欢迎的 Obsidian 组织方式。

### PARA

把所有笔记按用途状态分四类：

1. **Projects（项目）** — 有截止日期的事儿
2. **Areas（领域）** — 长期负责的领域（如"健康""财务"）
3. **Resources（资源）** — 未来可能有用的素材
4. **Archives（归档）** — 已完成/不再活跃的内容

**迁移技巧：** 从其他工具搬过来时，先把旧笔记全扔进 Archives，你就有了干净的桌面，但历史都还在。

### Zettelkasten（卡片盒）

核心三条：每篇笔记只写一个想法 → 用你自己的话重新表达 → 主动建立链接。

流程：**随手记 → 定期整理 → 重写并链接**。适合经常写作、做研究的人。它不是分类法，是写作法。

### 怎么选？

最好的方法是先写半个月，看哪种需求自然冒出来，再有意识地用对应方法。三种可以混着用。

## 七、插件推荐（按阶段安装）

### 7.1 通用安装方法

设置 → 社区插件 → 「关闭受限模式」→ 「浏览」打开插件市场 → 搜索 → 「安装」→ **点「启用」**（很多人忘了点启用以为插件坏了）

**如果插件市场打不开：** 切换网络、用加速工具，或去 [PKMer](https://pkmer.cn) 等国内镜像站手动下载插件文件，放到 `仓库路径/.obsidian/plugins/插件名/` 目录下重启 Obsidian。

### 7.2 第一批（立刻装）

1. **Calendar（日历）** — 右侧栏多一个可视化月历，能直观看到哪天写了日记、哪天没写。配合核心插件「日记」每天点击今天 → 写日记。

2. **Templater（高级模板）** ⭐ — 比核心模板插件强大得多。能在模板里自动填入当前日期、笔记标题、星期几，甚至弹出选择框让你选。
   - 常用变量：`<% tp.date.now("YYYY-MM-DD") %>`（日期）、`<% tp.file.title %>`（标题）
   - 实操建议：给最常用的几种笔记各做一个模板：日记、读书笔记、会议记录、项目笔记

3. **Image Toolkit（图片工具箱）** — 点击笔记里的图片弹出全屏预览，支持滚轮缩放、拖拽移动，装完即用无需配置。

### 7.3 第二批（用了一周后装）

4. **Tasks（任务管理）** ⭐ — 在任何笔记里写查询，自动汇总所有未完成任务。
   - 任务写法：`- [ ] 买牛奶 📅 2026-05-25`
   - 汇总视图：
     ````markdown
     ```tasks
     not done
     due before tomorrow
     ```
     ````
   - 实操建议：建一篇「任务总览」笔记，放几个不同条件的查询块

5. **QuickAdd（快速添加）** ⭐ — 把"灵感闪过 → 想记下来"的摩擦力降到最低。按快捷键 → 弹输入框 → 打字回车 → 内容自动追加到指定笔记，不超过 3 秒。
   - 典型配置：快速捕捉灵感 → 追加到「灵感收集箱」

6. **Periodic Notes（周期笔记）** — 补齐周记、月记、季记、年记。每周日花 15-20 分钟写一篇周回顾，坚持一个月你会惊讶自己原来做了这么多事。

### 7.4 第三批（用了两周后装）

7. **Kanban（看板）** — 在 Obsidian 里创建 Trello 风格看板（如「灵感池 → 写作中 → 已完成」），卡片内容可写 `[[笔记标题]]` 实现跳转。

8. **Dataview（数据视图）** ⭐ — Obsidian 最强大的插件之一，把笔记当数据库查询。给笔记加 frontmatter 后：

   ```markdown
   ---
   title: "《思考快与慢》"
   author: "Kahneman"
   rating: 5
   date-read: 2026-03-15
   tags: [心理学, 决策]
   ---
   ```

   查询示例：
   ````markdown
   ```dataview
   TABLE author, rating, date-read
   FROM "读书笔记"
   SORT date-read DESC
   ```
   ````

9. **Excalidraw（白板绘图）** — 手绘风格白板，画思维导图、流程图、概念图。图形里的文字可包含 `[[笔记标题]]` 双链。

10. **Various Complements（智能补全）** — 打字时根据仓库现有笔记标题弹出建议，按 Tab 补全。记不清完整标题时很有用。

### 7.5 安装节奏建议

**不是装得越多越好。** 先遇到问题，再找插件解决。装了不用等于没装。先把笔记写起来，所有插件都是为"写"服务的。

## 八、实用模板

### 1. 每日日记

启用核心插件「日记」即可，每天点日历上的今天 → 开始写。

### 2. 灵感收集箱

每次有想法就追加。配合 QuickAdd 一键追加。

### 3. 读书笔记

```markdown
# 书名

- **作者：**
- **读完日期：**
- **评分：**

## 核心观点

用自己的话写……

## 触动我的句子

## 行动清单

- [ ] 实践 xxx
```

关键：**不要抄原文，用你自己的话复述。**

### 4. 会议记录

标题：`2026-05-21 产品需求评审`。只记结论、决定、行动项，不用记每句发言。

### 5. 待办事项

`- [ ] 任务描述` — 完成时改成 `[x]`

### 6. 项目计划

```markdown
# 项目名

## 目标
## 关键节点
## 当前进展
## 问题/阻塞
```

### 7. 知识卡片（一篇一个概念）

能用简单的话讲清楚 = 真正理解了。讲不清楚说明还没搞懂，回去再学。

### 8. 收藏整理（替代浏览器书签）

每条写一句"为什么值得看"——不写理由的收藏永远不会再打开。

### 9. 周复盘

每周日一篇：

```markdown
# 2026 年第 21 周复盘

## 这周做了什么
## 最有价值的事
## 可以改进的
## 下周计划
```

### 10. 速查手册

把生活中反复要查的信息集中存一处：WiFi 密码、常用地址、报销流程、电脑故障处理等。`Ctrl/Cmd + O` 输入"速查"秒开。

## 九、从其他工具导入

- **Word：** 装 Importer 插件 → 命令面板「Importer: Import data」→ 选 Microsoft Word。少量文档可直接复制粘贴，标题/加粗/列表会自动转 Markdown
- **飞书文档：** 右上角导出 → 优先选 Markdown（没有就选 Word）
- **PDF：** 设置 → 文件与链接 → 附件默认存放路径 指定一个文件夹。直接拖进 Obsidian 编辑区，自动入库 + 生成链接
- **音频/视频：** 支持 mp3/wav/m4a/mp4/webm，文件放仓库里，用 `![[录音.mp3]]` 嵌入

## 十、AI 集成

### 推荐方案：DeepSeek + Copilot 插件

1. 安装 **Copilot** 插件（作者 Logan Yang），右侧栏出现聊天气泡图标
2. 去 [DeepSeek 官网](https://platform.deepseek.com) 注册拿 API Key（国内可用，不需要科学上网，价格极便宜）
3. 在 Copilot 配置中添加 Custom Model：
   - Name: `DeepSeek`
   - API URL: `https://api.deepseek.com/chat/completions`
   - API Key: 粘贴你的 Key

4. 在聊天面板输入"你好"测试，几秒内回复即成功

**其他备选方案：** 通义千问、文心一言、Kimi、OpenAI 等，配置方式相同。

**常见用法：** 选中笔记文字 → 让 AI 总结、翻译、改写、写大纲；或直接问关于你笔记的问题。

## 十一、备份

**底线：至少保留一份独立的本地备份。**

推荐方式：
- 每周 rsync 到外置硬盘
- 配合 Git 做版本管理（仓库根目录 `git init`）
- 云同步：Obsidian Sync（付费）或配合 iCloud / Dropbox / Syncthing 等

## 十二、FAQ

**Q：数据安全吗？会不会丢？**
笔记是本地 .md 文件，不依赖任何云端，软件公司倒闭也能用文本编辑器打开。但本地文件丢了就真丢了——做好备份。

**Q：不懂 Markdown 怎么办？**
实时预览模式下几乎感觉不到在写 Markdown，三天就全记住了。

**Q：旧笔记能导入吗？**
Notion、印象笔记、Bear、Apple Notes 都有迁移方案或社区插件。但建议先用 Obsidian 建立新习惯，再慢慢迁移旧笔记。

**Q：社区插件安全吗？**
开源，官方有基本审核。建议只装下载量大、评价好、更新频繁的插件。

**Q：MOC、PARA、Zettelkasten 我该选哪个？**
不是非要选。先写半个月，看哪种需求自然冒出来再说。三种可以混着用。

## 十三、心法

1. **别贪快。** Obsidian 的价值随时间累积。一周写 10 篇可能没感觉，坚持三个月，你会真的拥有"第二大脑"
2. **不要花几小时设计完美分类** — 没写过几篇笔记就开始设计分类，必翻车
3. **先写起来** — 一篇日记、一个想法、一句话都行
4. **文件夹不超过 1-2 级** — 靠链接和标签做主组织
5. **数字前缀让排序固定** — `01-xxx`、`02-xxx`
6. **拿不准的笔记先丢"00-收件箱"**，定期清理

从今天开始就好。慢慢积累，Obsidian 会变成你的终极武器库。
