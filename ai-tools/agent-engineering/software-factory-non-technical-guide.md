# 什么是软件工厂(Software Factory)？——非技术领导者的完整指南

> **来源：** [@businessbarista](https://x.com/businessbarista/status/2061573150719021542) | Tenex Labs CEO
> **阅读：** 93K+ 浏览 | 217 赞 | 640 书签

---

2023 年大家都在聊 ChatGPT，2024 年是 GenAI，2025 年是 Agent，2026 年先是 OpenClaw 火爆，现在目光转向了 **Software Factory（软件工厂）**。

有些公司宣称 90% 的生产代码由 AI 编写，顶级 IC 的产出抵得上过去 20 人规模的工程团队。但现实是：**大多数公司离真正的 Software Factory 还差得远。**

这篇长文来自 Tenex Labs（为企业搭建 Software Factory 的公司）CEO，从非技术领导者的视角，系统性地拆解了什么是 Software Factory、为什么突然火起来、以及你的组织现在处在哪个阶段。

---

## 什么是 Software Factory？

去掉行话，核心很简单：

**Software Factory = 软件开发从"手工艺"变成了"生产线"**

- **手工艺模式（Craft Model）**：人坐下来手写代码。就像定制手工车，灵活但慢、不稳定。产出随人数线性增长。
- **工厂模式（Factory Model）**：工作被工业化。像现代汽车制造厂：更快、更可靠、可扩展，但需要前期投入建立流程和工具。

**最极端版本**来自 StrongDM 的两条原则：
> "代码不能由人编写。"
> "代码不能由人审查。"

——这不是说人类不参与，而是**人类从执行者升级为指挥者**。工厂不是"人用 AI 更快写代码"，而是"人不再直接写代码"。

---

## 为什么这个术语突然流行？

三个因素同时爆发：

**1. 实在的证明摆在那了。**
@ryancarson 运营着他所谓的"Code Factory"——Agent 写代码、审查、跑测试、排查错误、监控生产，他只负责设定护栏和目标。

**2. 大公司给这个词背书。**
Microsoft 开始在推"Agent Factory"作为软件构建的新范式。大厂采用某种语言，它就迅速进入主流。@chamath 经常谈 AI 改写软件生产函数。

**3. "XX% 代码由 AI 编写"变成了 CEO 间的攀比指标。**
不是最合适的炫耀指标，但确实让 agentic engineering 获得了关注。

## Software Factory 阶梯（五层模型）

要构建 Software Factory，你首先需要定位自己工程组织现在在阶梯的哪一层。**五层，0-4 级。** 应该在一分钟内能判断出来。

为了具体化，每一层都跑同一个场景：**一个客户遇到 bug——"下单"按钮不工作了。** 注意人和 AI 的角色如何随层级变化。

### Level 0: Artisan（工匠）

客户邮件联系客服 → 工程师听说 → 复现 → 手写修复 → 另一个工程师读代码审查 → 人工测试 → 人工部署上线

**一切由人驱动。** 软件是人手工制作的，和生产几百年前的铁匠铺没有本质区别。

### Level 1: Assisted（辅助）

步骤完全一样，只是工程师用 AI copilot **稍快一点**地写修复代码。产出略有提升，但流程没变：**人仍然做所有事，人仍然是每一道门禁的守卫。**

### Level 2: Delegated（委派）

工程师把 bug 交给 agent。Agent 自己写修复、自己开 PR。**但人仍然审批每一个 PR 后才能合并。** 有了真正的杠杆效应——一个工程师可以引导多个 agent。

**关键门槛：从"人写代码"到"人指挥 agent 写代码"。**

### Level 3: Supervised Factory（监督式工厂）

Agent 监控生产 → 捕获错误 → 自动分类。编码 agent 自己写修复，另一个 agent 审查。测试自动跑。如果风险低且所有检查通过，**修改直接上线没有人参与。** 人只在例外情况下介入——高风险变更或需要架构决策。

**关键门槛：低风险变可以无人工审批自动上线。**

### Level 4: Autonomous Factory（自主工厂）

bug 被捕获、修复、审查、测试、部署——**团队里没有任何人知道发生过这件事。** 人类不在"看守生产线"，他们在决定"让生产线生产什么"。

## 快速定位你的组织在哪一层

问这 11 个问题，几分钟就清楚：

1. **我们的 PR 中有百分之多少在没有人工审阅的情况下上线？**
   - 0% = Level 1 或更低

2. **修 bug 或小功能时，第一版是谁写的？**
   - 人写的 = Level 0 或 1
   - 人指挥 agent 写的 = Level 2+

3. **Agent 能自己开 PR 吗，还是每次变更都得人写？**
   - 后者 = Level 1 上限

4. **我们有规则让低风险变更自动上线、高风险升级给人吗？这些规则谁设的？**
   - 这是 Level 3 的核心

5. **生产环境出了事，第一个知道的是人还是系统？谁在做分类？**
   - 人知道 = 早期；系统捕获并分类 = Level 3

6. **每次变更都自动跑测试吗？系统能在无人介入时阻止坏的变更吗？**
   - 告诉你质量门禁是不是真实的，还是仅仅是"人点一下 approve"

7. **如果 agent 上线了错误的东西，我们能多快发现并回滚？**
   - 你的安全网决定了你安全地爬到多高

8. **我们的 AI 工具能安全访问自己的代码库、文档和系统吗，还是只能依赖通用的公开知识？**
   - 通用知识 = 你被上限卡住了，直到解决这个问题

9. **我们有 agent 可以接入的内部平台吗，还是每个工程师自己搭自己的工具？**
   - 据 DORA 报告，这是单一最大的区分因素

10. **如果我明天拿走所有人的 AI 工具，我们流程会变吗，还是只是变慢？**
    - 只是变慢 = Level 1 的强化版自动补全
    - 流程会崩 = 你真正工业化成功了

11. **瓶颈现在在哪：写代码，还是审查和上线？**
    - 还在写代码 = 早期
    - 瓶颈移到了下游 = 成熟
    - **这个迁移就是整个意义所在**

## 如何爬梯子

### Level 0 → 1：让每个工程师用上 AI

这一步更多是组织问题，不是技术问题。给每个工程师一个编程助手，明确哪些工具是被认可的，鼓励实验而不只是放任。让工具成为标配，而不是可选。

### Level 1 → 2：让 Agent 拥有完整任务

现在工程师只是用 AI 打字更快——这是天花板。**下一步是交给 agent 一个完整的工作单元。** 从你最成熟测试、风险最低的 ticket 开始。明确界定："这个由 agent 从开始做到结束，我负责审查方向，不是审查每一行代码。"

**关键指标**：agent 能自己开 PR。

### Level 2 → 3：消除审查瓶颈

现在瓶颈是你——人在审批每一个 PR。**关键动作：让低风险变更在没有人在回路中的情况下上线。** 具体操作：
- 写下"什么是低风险"的明确标准
- 配置测试管道，在满足标准且测试全过时自动合并
- 设定人工上报规则：当高风险或不明确时，升级给人

**这不是扔掉质量——是把质量门禁自动化。**

### Level 3 → 4：移除最后一个人类触发器

Level 3 还是由人启动工作。Level 4 是 agent 监控生产 → 捕获 bug → 自己开 ticket → 跑整条线。人在设计、决策和监督例外，不在启动每一单工作。

## 你公司的哪个功能是下一个工厂？

这个问题是作者目前最热衷的方向。

瑞安·卡森说得对：**工程只是第一个工厂。下一步是"公司工厂(Company Factory)"。**

但不是每个功能都同样准备好了。决定性因素就一个——**可验证性(Verifiability)**。

@karpathy 给出了最好的框架：传统软件自动化的是你**能指定的**；AI 自动化的是你**能验证的**。如果一个任务有自动的成功信号，机器可以反复练习，你就可以信任它的输出。

**一个功能"工厂就绪"的程度取决于：**

| 维度 | 说明 |
|------|------|
| ✅ **可验证的输出** | 能否自动检查工作的对错？这是最关键的单一因素 |
| ✅ **数字化输入输出** | 工作发生在软件中，不是握手或物理房间里 |
| ✅ **高频率和重复性** | 有足够多的重复任务值得搭建一条生产线 |
| ✅ **可逆性** | 当机器出错时，能低成本捕获并撤销吗？ |

**按这个框架排序：**

🟢 **最工厂就绪（优先建）：**
- 软件工程
- QA
- 数据管道和分析
- DevOps
——这些都有 ground-truth 检查：测试通过、schema 验证、仪表盘能对账

🟡 **逐渐就绪的：**
- 财务和会计的部分工作（对账有标准答案）
- 客户支持（解决了还是没解决，CSAT 可衡量）
- 销售和营销的操作层面（发出了没有）

🔴 **最不工厂就绪（暂时）：**
- 战略制定
- 设计品味
- 新颖架构
- 高管判断
- 关系和信任建立
——任何"对错"是主观的或要很久才知道结果的事

## 如果只记住五件事

1. **Software Factory 不是"工程师用 AI"。** 它是 agent 写、审、测、上线的生产线，人类在设计生产线而不是在工作。
2. **别再吹嘘"XX% 代码由 AI 编写"。** 问真正的问题：你生产线的多少环节不需要人在里面？
3. **五层阶梯：** Artisan（一切都是人）→ Assisted（强化版自动补全）→ Delegated（agent 驱动执行）→ Supervised Factory（人负责方向和例外）→ Autonomous Factory（人在设计线，不在线上）。
4. **爬梯子的方式：一次移除一个人的环节，速度取决于你的安全网**（测试、版本控制、内部平台）有多可靠。
5. **工程只是第一个工厂。** 下一个是哪个有可验证、数字化、高频、可逆工作内容的职能。在竞争对手之前找到它。
