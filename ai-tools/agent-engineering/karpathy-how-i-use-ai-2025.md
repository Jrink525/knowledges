# Karpathy: How I Use AI (2025) — 实践指南总结

> **原文**：Andrej Karpathy 的 ~2 小时 AI 实操演示视频（2025 年发布）
> **来源**：X/Twitter @0xchromium 分享
> **记录时间**：2025-06-23
> **类型**：AI 工具使用指南 / LLM 实操

## 核心框架

Karpathy 提出的核心思维模型：

**💡 你是在和一个「zip 文件」对话**
- 预训练 = 把整个互联网压缩成一个有损的 zip 文件（神经网络参数）
- 1TB 参数 ≈ 1 万亿参数
- 知识截止日期 = zip 文件的压缩时间点
- 后训练（post-training）= 给这个 zip 文件加个笑脸，规范输出风格和形式
- 所以遇到模型拒绝说"我不知道"时，记住它是 zip 文件，不是搜索引擎

## 一、模型选择：三个维度

### 1. 大小模型
| 类型 | 特点 | 适用场景 |
|------|------|---------|
| 大模型（如 GPT-4o） | 知识渊博、写作创意好 | 复杂问题、高质量写作 |
| 小模型 | 响应快、但易幻觉 | 简单查询、快速任务 |

### 2. 思考模型 vs 非思考模型
- **非思考模型（GPT-4o 等）**：快，适合日常使用
- **思考模型（o1/o3/R1 等）**：通过强化学习训练，会输出思维链，需要等数分钟
  - OpenAI: o1, o3-mini, o1-pro
  - DeepSeek: R1（Perplexity 上可看原始思维链）
  - Grok: "Think" 按钮切换
- **策略**：先用非思考模型，遇到复杂问题（数学、代码、推理）再切思考模型
- 旅行建议等简单任务不需要思考模型

### 3. 工具能力
- **搜索**：需要最新信息时用（Perplexity 是最好的搜索 LLM）
- **Python 解释器**：数据可视化、生成图表
- **代码执行（Artifacts）**：Claude 的 Artifacts、ChatGPT 的图表生成

## 二、搜索工具的使用

Karpathy 推荐 **Perplexity** 作为默认搜索工具（优于 ChatGPT 的内置搜索）

- 搜索场景：当答案可以通过 Google 搜索 + 访问几个顶部链接获得
- 适合：最新信息、变动频繁的信息、产品评测
- 对比测试：Claude 无搜索功能 → 无法回答需要实时信息的问题
- Gemini 2.0 Pro 也没有搜索功能（实验性）
- GPT-4o 有搜索但搜索结果可能不是最新的

## 三、深度研究（Deep Research）

ChatGPT Pro（$200/月）功能，结合搜索和思考能力：
- 自动进行数十个搜索，阅读多篇论文
- 输出带引用来源的详细研究报告
- Karpathy 用例：研究 Brian Johnson 的长寿补剂成分
- 等待时间约 5-15 分钟

## 四、编程实操

### 4.1 日常开发：Cursor（或类似 IDE）
- **不是** 手动复制粘贴代码到 ChatGPT，而是用 Cursor 等 IDE 集成
- Cursor 底层调用 Claude 3.7 Sonnet API
- 三个进化阶段：
  1. **Ctrl+K**：修改单行代码
  2. **Ctrl+L**：解释代码块
  3. **Ctrl+I（Composer）**：全自动 AI agent 编辑多文件

### 4.2 Vibe Coding（氛围编程）
- Karpathy 创造的术语
- 完全交给 AI agent 主导，你只需用自然语言描述需求
- 例子：用 Cursor + Claude 几分钟内完成井字棋 React 应用
- 心态：坐看 AI 写代码，做"监督者"而非"实现者"

### 4.3 可用的编码辅助工具
- **Claude Artifacts**：React 组件直接在浏览器中渲染
- **ChatGPT Code Interpreter**：数据分析和图表生成
- **Cursor**：专业 IDE 集成，可操作本地文件

## 五、多模态交互

### 5.1 语音模式（Karpathy 强调：**语音是最好的输入方式**！）

**假语音（Fake Audio）**：
- Speech-to-Text + Text-to-Speech 管道
- 移动端：80% 查询通过语音（手机键盘太慢）
- 桌面端：用 Super Whisper / Whisper Flow / Mac Whisper 等系统级转录
- **核心建议：不要打字，用语音！**

**真语音（True Audio / Advanced Voice Mode）**：
- 音频直接在 LLM 内本地处理（音频 → token → 推理）
- 模型可以理解语气、情感、语速
- OpenAI 的 Advanced Voice Mode
- ChatGPT 中的音频图标按钮

### 5.2 图片输入
- 营养标签解析：Karpathy 用 AI 读取补剂的成分表
- 血检报告分析：让 AI 理解医学指标
- 数学题拍照解题

### 5.3 视频输入
- LLM 可以通过视频帧理解视频内容
- 实时视频问答（如识别物品、评析电影片段）

## 六、自定义设置

### 6.1 Custom Instructions
- Karpathy 自己的设置例子：
  - 学习韩语 → 指定敬语格式
  - 设置输出语气和风格偏好

### 6.2 Memory 功能
- 模型记住用户偏好和个人信息
- 跨会话持久化

### 6.3 Few-shot Prompting
- 给出输入和期望输出的示例
- 比单纯描述格式要求效果好得多

## 七、基础设施心智模型

Karpathy 总结的 AI 交互七大维度：

```
┌──────────────────────────────────────┐
│           LLM 应用评估框架            │
├──────────────────────────────────────┤
│ 1️⃣ 模型层（zip 文件本身）            │
│   - 知识截止日期                      │
│   - 模型大小（参数规模）              │
│   - 定价层级                          │
│                                       │
│ 2️⃣ 推理层（思考模式）                │
│   - 非思考模式（快速响应）            │
│   - 思考模式（深度推理）              │
│                                       │
│ 3️⃣ 工具层                            │
│   - 互联网搜索                        │
│   - Python 解释器                     │
│   - Artifacts/代码执行                │
│                                       │
│ 4️⃣ 多模态层                          │
│   - 文本 ↔ 语音（真/假）              │
│   - 图像 ↔ 文本                       │
│   - 视频理解                          │
│                                       │
│ 5️⃣ 上下文窗口                        │
│   - Token 是珍贵资源                  │
│   - 新会话 = 清空上下文               │
│   - 不要塞无关信息                    │
│                                       │
│ 6️⃣ 质量功能                          │
│   - 文件上传                          │
│   - Memory 功能                       │

│                                       │
│ 7️⃣ 平台层                            │
│   - Web 端 vs 移动端特点差异          │
│   - 功能可用性检查                    │
└──────────────────────────────────────┘
```

## 八、关键行为建议

### Do ✅
- **用语音代替打字**：50-80% 的查询可以靠语音完成
- **根据任务选择模型**：日常用 GPT-4o，复杂问题才切思考模型
- **用 Perplexity 做搜索**：比 Google 好，比 ChatGPT 搜索成熟
- **用 IDE 集成（Cursor）编码**：别复制粘贴，用 Composer 多文件修改
- **先试非思考模型**：不够好再换思考模型
- **Few-shot 示例**：给出输入输出示例比描述要求有效

### Don't ❌
- 别把上下文窗口塞满无关信息
- 别在简单查询上浪费思考模型（等几秒和等几分钟的区别）
- 别手动搜索 → 让模型用搜索工具
- 别复制粘贴代码 → 用 Cursor/Claude Code

## 九、竞品生态速览

| 产品 | 优势 | 特点 |
|------|------|------|
| ChatGPT | 最全功能、原厂 | 搜索、代码、多模态、GPTs |
| Claude | Artifacts、编码能力 | 前端渲染、Cursor 底层 |
| Perplexity | 搜索最优 | 集成 R1 思维链显示 |
| Gemini | Google 生态 | 多模态理解、推理 |
| DeepSeek | 开源、R1 思考模型 | 透明思维链 |
| Grok | X 实时数据 | 幽默不设限，Think 按钮 |
