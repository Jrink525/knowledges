# 如何做好研究

> **来源：** [如何做好研究 — Vivek (@itsreallyvivek)](https://x.com/itsreallyvivek/status/2064686372737454155)
>
> 作者是 Anthropic 和 MATS Program 的研究员，曾在 Mayo Clinic 和 Starbucks 任职。

---

没人真正教你做研究。你拿到一张桌子、一个别人选好的问题、以及一条模糊的指令——做出点新颖的东西。所以大多数人只能从他们能看到的东西（论文、推文、公告）来反向推导"研究员这份工作要怎么做"，结果学到的是怎么装得像研究员，而不是怎么成为研究员。真正的技能是一堆小技能的叠加，而几乎每一条都可以被刻意训练。

---

## 1. 自己选问题

贝尔实验室的 Richard Hamming 有个习惯，让他在午餐时间不受欢迎：他会问坐在旁边的人，他们领域里什么问题是重要的，接着问为什么不干那个。人们纷纷换桌。这个问题刺痛我们，因为大部分人答不上来。我们不是选择问题，而是吸收问题——从导师那里、从某大实验室上个季度的 announcement、从那篇所有人都在转发引用的论文里。

吸收来的问题有一个麻烦：你拿到了结论，却没有背后的推理过程。你知道某个著名实验室关注某个方向，但不知道为什么、不知道他们期望找到什么、不知道什么事情会让他们放弃这个方向。等他们转向了，你一年后才知道。而且在一个已经火热的问题上，你是在跟一千个起步更早、算力更多的人赛跑。

John Schulman 的 ML 研究指南把工作分成两种模式：一种是你读了文献然后找可以改进的东西；另一种是你选一个你真心希望存在的结局，然后反向推导出需要的实验。他推崇第二种，安静的底层的理由是——这种方式能制造原创性。一个你真正在意的目标，会把你拖进没有任何综述论文覆盖过的地带。

至于"品味"，人们总把它说成一种天赋。它更像一块肌肉。每次跑实验前先预测结果；遮住论文的结果部分，仅凭方法去猜数字；记下本月发布的哪些东西两年后还会被提起，然后回过头检查你的命中率。一次预测加一次校正，重复几百次——这就是所有好的模型被训练的方式，包括你脑子里那个。

## 2. 升级你的输入

共享的阅读清单产生共享的想法。如果你的信息源只是 arXiv 的热门页面加上群聊里幸存下来的东西，那你总会跟其他人在同一时间得出同样的结论——这让那些结论大约一文不值。

旧材料的价格被严重低估了。这个领域总在延迟播放自己的过去：MoE（混合专家模型）可以追溯到 1991 年，LSTM 到 1997 年，反向传播在 1986 年才大规模普及。Rich Sutton 在 2019 年只用了大约一千个单词写就了《苦涩的教训》（The Bitter Lesson），但它对领域走向的预测比十倍长度的综述都准。Claude Shannon 在 1952 年做过一次关于创造性思维的演讲，他的开场动作是把问题缩小到几乎微不足道，攻克小版本，然后逐步把难度重新引入。就这一招，能帮你穿越的墙壁比所有现代生产力建议加起来都多。

广度跟深度一样重要。可解释性无耻地借鉴了神经科学；评测设计本质上披着白色实验服的机制设计；对 GPU 到底怎么搬运内存有一点点感觉，就能让你在 benchmark 还没跑之前就知道哪些架构论文注定失败；而诚实的统计学可能是 ML 领域最稀缺的技能——大量号称"严谨"的发表成果不过是带误差棒的玄学。

还有一件事：读论文本身，而不是读那些总结论文的推文。附录才是尸体埋藏的地方，而 limitations（局限性）章节通常是整篇论文最诚实的一段。

## 3. 全部写下来

Paul Graham 指出，一个想法可能在你试图把它落到纸上之前感觉完全成型。纸张找到了你脑子掩盖的那些漏洞：你从未测试过的假设、逻辑上其实不成立的步骤、两个悄声矛盾的主张。

费曼的规则是：你第一个必须避免欺骗的人是你自己，因为你是最容易骗的目标。写作是迄今为止最便宜的防御手段。达尔文更进一步，把写变成了流程：任何与他理论相悖的事实都立即写下来，因为他发现自己大脑删除不利证据的速度比删除有利证据更快。你的记忆对你的失败实验也是这样。保持日志：假设、设定、预期、结果、更新后的认知。重读上个月的记录是一件比任何审稿人的意见都更能让你谦虚的事。

然后把一部分写的东西公开。Olah 和 Carter 的《研究债务》（research debt）一文指出了领域被"未消化的想法"堵塞的问题，清晰的解释本身就是一种真正的贡献，而不是一个服务性工作。今天很多做可解释性的人是通过可读的博客文章而不是会议论文进入这个领域的。公开发布的文字也是一种最强的资历证明——它是一份无法伪造的"你是如何思考的"样本。

---

## 4. 缩短循环

那些关于 Alec Radford 的故事很少涉及一次天才的灵光一现。它们涉及的是量：每天跑的实验更多，每周抛掉的错误想法更多，一个比任何人都更快更新的现实模型。这才是真正的游戏。研究速度很大程度上就是发现自己错了的速度。

这使工具成为了一等一的研究活动。启动一次运行应该是一条命令。绘图也应该是。每个实验都应该能从其配置复现，比较两次实验应该是几秒的事情，而不是花一下午考古。Karpathy 的训练神经网络的食谱里有一个步骤，回报率能翻上百倍：在大规模训练之前先在一个 batch 上过拟合。30 秒，一半的 bug，消失了。把一切缩小到便宜为止，搞对了，再花钱上算力。

别再觉得工程是研究的小弟。在前沿，这两件事已经融合了。那些能自己搭实验框架、写评测、建数据管线的研究员，是那些假设真正能被验证的人。其他人都在排队等着。

---

## 5. 盯着输出看

一条下降的 loss 曲线不是分析，是安慰。你的实验抛出的信息远超你消费的：转录、失败案例、分布中奇怪的那条尾巴。大部分死在一个日志文件夹里，无人阅读。

Karpathy 的食谱在写任何训练代码之前就开始：花数小时手工看原始数据。大多数 ML bug 活在数据里，而且它们静默地失败——什么都不崩溃，你只是得到一个平庸的模型和一个关于为什么的错误理论。

Andrew Ng 教了一个不那么好看的动作超过十年了，因为没别的比得上它：拉出一百个失败案例，全部读完，分类成堆，攻击最大的那堆。它在模型上管用，在评测上也管用——一个你从未读过转录的 benchmark，是你根本不理解的 benchmark。一个真正奇怪行为的转录教给你的东西，比下一个百分点的准确率永远多。

---

## 6. 故意漫游

你的第一个子领域是时机上的偶然，所以就像对待偶然一样对待它。花真实的时间在可解释性、评测、强化学习、系统——然后再决定你住在哪里。这个领域总有一个角落，你独特的怪异是一种不公平的优势，找到它的唯一方法就是在好几个地方付学费。没有谁能免掉这笔学费。

先跑每个想法的可丢弃版本，让大多数想法早死。调 baseline 调到痛为止——因为 ML 的墓地里满是在一个调好的 baseline 面前蒸发掉的"提升"，而审稿人是这个点上学到教训的最差人选。做 ablation 做到知道哪个组件带来了结果。通常只有一个，而且通常不是标题里说的那个。

广度也是一种保险。子领域会饱和，它们全部都会，通常是在它们在 Twitter 上达到峰值之后。那些能在这些转折中持续产出的人，是那些已经知道如何在邻近领域里走动的人。

---

## 7. 找到你的人

Hamming 注意到一个模式——最终做出重要工作的人是什么样的：关着办公室门的同事在任意一年中产出更多；但开着门的同事做的事情才算数——因为那些"打扰"携带了关于世界真正需要什么的信息。你的"敞开的门"大概是你的收件箱。保持这样。

在研究领域，慷慨的复利效应是任何别的东西都无法比拟的：复现一个结果并公布你的发现；把你为自己打造的工具开源；用通俗的语言解释一件困难的事情。回报会在几个月后从旁边冒出来——作为一个合作机会、一个引用、一个你本来无法申请的职位。半成形的想法也先公开发出来——因为在时间线上犯错比在印刷品上犯错便宜得多。而那个在你沉入三个月之前告诉你这个想法是垃圾的合作者，比算力还值钱。这种关系买不来，只能挣得。

---

## 8. 长期游戏

巴斯德说"运气偏爱有准备的头脑"，而 Hamming 在上面建立了一整套职业哲学：知识和生产率像利息一样复利。每天的边缘增益单独看微不足道：你读什么、你记什么、你的循环跑多快、你跟谁争论。给它们几年时间，它们会产出那种从外面看像是运气的职业生涯。在感觉还没必要的时候就复利。未来的你已经知道这是便宜的部分了。

---

### 核心要点

| 原则 | 一句话总结 |
|------|-----------|
| **自己选问题** | 别吸收别人的问题；选一个你真心想要的结果，反向推导 |
| **升级输入** | 读旧材料、跨界、读原文胜过读摘要和总结帖 |
| **全部写下来** | 写作暴露思想的漏洞；公开写作是无可伪造的思考样本 |
| **缩短循环** | 研究速度 = 发现自己错了的速度；工程不是二等公民 |
| **盯着输出** | Loss 曲线不是分析；读失败案例比读准确率更有用 |
| **故意漫游** | 你的第一个子领域是偶然；交学费探索，再决定 |
| **找到你的人** | 保持开放；慷慨复利；公开半成形想法能省三个月 |
| **长期游戏** | 每天微小增益的复利会在几年后在别人看起来像运气 |
