## 1. SkillOpt — 将 Skill 文档当作可训练参数

**Microsoft Research** | [arXiv](https://arxiv.org/abs/2605.23904) | [GitHub](https://github.com/microsoft/SkillOpt) | [项目页](https://microsoft.github.io/SkillOpt/)

### 核心思想

大多数工程师手写 agent skill 文档（如 SKILL.md），寄希望于它能够泛化。SkillOpt 认为：**文档本身应该像参数一样被优化**——将自然语言 skill 文档作为冻结 agent 的"可训练外部状态"，通过 rollout、反思和有界编辑来学习。

### 关键设计

- **Skill 文档 = 训练参数**：一个 optimizer 模型对 skill 文件提出增/删/替换的有界编辑，只当通过 held-out 验证集时才接受
- **Textual Learning Rate**：编辑预算控制每轮重写的激进程度，batch 和 momentum 在文本空间中实现而非梯度空间
- **验证门控**：每次编辑必须通过 held-out 检查才能保留，把 skill 编写变为**可测量的优化循环**

### 实验结果

- **52/52 全胜**：在 6 个 benchmark 和 7 个目标模型上击败 Trace2Skill、TextGrad、GEPA、EvoSkill、手写 skill 和 one-shot skill
- GPT-5.5 直聊 +23.5 分，Codex 循环 +24.8 分，Claude Code +19.1 分（相比无 skill 基线）
- 消融实验证实：有界编辑、验证门控、拒绝缓存三个组件都至关重要

### 为什么重要

> 如果 skill 文档是你优化的对象，瓶颈就从基础模型能力转移到 **"你能多好地训练冻结 agent 周围的自然语言状态"**。这是一个廉价、模型无关的杠杆，大多数团队还没用上。

## 2. Compiling Agentic Workflows into Weights — 将工作流编译进权重

**「Subterranean Agent」** | [arXiv](https://arxiv.org/abs/2605.22502)

### 核心思想

完整的 agentic workflow 可以**蒸馏进小模型的权重**中，推理成本降低约两个数量级，同时保持接近前沿的任务质量。

### 关键设计

- 编译的不只是答案，而是**整个工作流**：多步 LLM 调用、工具调用、中间 scratchpad、决策节点
- **协调器溶解进模型**：传统 agent 框架每请求运行一次 planner 循环，编译进权重后消除每次调用的编排开销
- 达到原始工作流质量的 **87-98%**，成本 **128-462× 更低**

### 为什么重要

> 大多数生产 agent 每天数千次地重复支付编排循环的成本。如果这个循环可以**一次性编译进廉价模型**，部署 agent 系统的经济学将根本改变，特别是对高吞吐量的窄域工作流。

## 3. AutoScientists — 去中心化自组织科研团队

**Harvard** | [arXiv](https://arxiv.org/abs/2605.28655) | [GitHub](https://github.com/mims-harvard/AutoScientists)

### 核心思想

一个**完全去掉中央规划器**的去中心化 AI agent 团队。agent 围绕有希望的假设自组织、在消耗实验算力前互相评审提案、记录成功与失败以避免重复探索。

### 关键设计

- **无中央规划器**：agent 解读共享实验状态，围绕有希望的方向组建团队，进度停滞时重组
- **先评估再动手**：提案在分配任何实验算力前被评审打分，减少浪费的试验
- **失败共享**：失败和成功都记录，个体 agent 不会重走已被探索过的死胡同

### 实验结果

- BioML-Bench（24 个生物医学 ML 任务，涵盖成像、蛋白质工程、单细胞组学、药物发现）：**74.4% 平均排名百分位**，比最强先验 AI agent **提升 +8.33%**
- GPT 优化速度快 **1.9×**，ACE2-Spike 任务 Spearman 提升 **+12.5%**

### 为什么重要

> 大多数多 agent 研究系统仍然通过一个成为瓶颈的规划器漏斗决策。**去中心化自组织 + 显式失败共享**为长期科学搜索提供了不同的蓝图，并在困难的生物医学基准上站住了脚。

## 4. Language Models Need Sleep — 语言模型需要睡眠

### 核心思想

注意力机制随上下文长度平方增长，长程 agent 的上下文成本不断膨胀。这篇论文研究了一种**类睡眠的整合机制**：模型定期将近期上下文转换为持久化的快速权重（fast weights），然后清空 KV cache。

### 关键设计

- 近期上下文折叠进 SSM 块的**快速权重**，然后丢弃 KV cache
- 额外计算在"睡眠"阶段离线完成，"清醒"时预测保持低延迟
- **越难的任务，睡眠帮助越大**：需要在长历史中进行最复杂推理的任务获益最大

### 为什么重要

> 长程 agent 是最先感受到上下文平方增长成本的系统。这个受生物启发的整合步骤为**越来越长的上下文窗口**提供了一个有原则的替代方案，而且自然地映射到已经用于效率的 state-space 架构上。

## 5. Adapting the Interface, Not the Model — 适配接口，而非模型

**「Life-Harness」**

### 核心思想

冻结的 LLM agent 在确定性规则环境中反复失败，你**需要重新训练模型吗**？Life-Harness 说：不需要。许多失败来自模型-环境接口的不匹配，修复属于运行时的 harness（支架）。

### 关键设计

- 重复错误转化为**可复用的运行时修复**：动作实现、环境契约、轨迹调控、过程性技能四个领域
- **模型冻结、环境不变**：只改接口层，适合任何 backbone，避免微调的风险和成本
- 跨 7 个确定性 agent benchmark + 18 个模型 backbone：**116/126 设置提升**，**88.5% 平均相对改进**

### 为什么重要

> 这是 **"代码即 haress"** 论点的更多证据：大部分 agent 失败是接口问题，可以用 harness 工程修复而不需要重新训练。对于构建者，杠杆在运行时，不在模型。

## 6. The Efficiency Frontier — 上下文效率前沿

### 核心思想

上下文成本主导生产 LLM 账单，正确的策略取决于**预处理被重复使用的频率**。这篇论文将上下文策略选择建模为一个感知部署的优化问题，联合考虑任务性能、token 成本和重用率。

### 关键发现

- **重用感知成本模型**：参数化的对数效用指标，对更多上下文施加递减收益，同时分摊预处理成本
- **不同的运作区间**：检索策略和预处理策略之间**有清晰的转换边界**，哪个获胜取决于你重用预处理上下文的次数
- 在 5000 个 HotpotQA 实例上，部署感知优化将有效 token 使用减少约 **25%**，压缩达到 **>50% 更低的 token 成本**

### 为什么重要

> 大多数团队一次选择上下文策略然后每个请求都为其付费。**将上下文管理视为显式的成本-性能优化**，把一个猜测变成可测量的决策，常见工作负载上就有两位数的节省。

## 7. Forecasting Scientific Progress with AI — AI 预测科学进展

**「CUSP」**

### 核心思想

前沿模型能预测科学走向吗？CUSP 是一个基于 **4,760 个真实科学事件**、跨多学科、每个事件都针对已验证的知识截止日期构建的 benchmark。

### 关键发现

- **识别 ≠ 预测**：模型可以从竞争选项中识别出有希望的研究方向，但**不能可靠预测成果是否真的会实现**，也系统性地错误估计时间
- **领域依赖，时间预测最难**：AI 进展的时间可预测性 > 生物、化学、物理。时序预测是全面最弱的能力
- 不是简单的训练截止人工产物：性能对事件是否在模型训练截止前后**基本不敏感**
- **系统性过度自信和响应偏差**：不确定性估计不可靠

### 为什么重要

> 随着实验室依靠 AI 来筛选研究赌注，CUSP 提供了一个受控的方式来衡量 AI 在**哪里有帮助（浮出方向）** 和 **哪里失败（预测结果）**。

## 8. AgingBench — 你的 Agent 也在老化

### 核心思想

**AgingBench** 是一个面向 agent 生命周期工程的纵向可靠性 benchmark。长期运行的 agent 仍然像刚初始化的模型一样被评估。

### 四种老化机制

| 老化类型 | 描述 |
|---------|------|
| 🔄 **压缩老化** | 写入时摘要丢弃了未来相关的细节 |
| 🔄 **干扰老化** | 积累的相似记忆挤掉了目标事实 |
| 🔄 **修订老化** | 变更或派生的状态未被正确更新 |
| 🔄 **维护老化** | 常规生命周期事件导致的退化 |

使用**时序依赖 DAG** 对跨会话结构建模，产生的是操作生命周期上的老化曲线，而不是单日的得分。

### 为什么重要

> 长期运行的 agent 会老化，但学术界和工业界还主要在评估"第一天"的冷启动性能。AgingBench 提供了系统性地衡量和修复 agent 退化的方法论。

## 9. Harnesses Are Not Uniformly Better — 支架不是越精细越好

### 核心思想

通过推理时轨迹对齐的视角研究 LLM agent harness。harness 分离为两种机制：**任务分解**（将任务结构化为子目标）和**引导执行**（在执执行期间重塑局部动作分布）。

### 关键发现

**更精细的 harness 并非一致更好**。增加分解或引导可以改善执行，但也可能降低最终成功率，产生具体失败模式：
- **过度分解**（over-decomposition）
- **过度修剪**（over-pruning）
- **幻觉执行**（hallucinated execution）

**令人惊讶的发现：** 只指定初始步骤、其余留给 agent 的**部分 harness**，可以达到比完全结构化工作流更高的通过率。

### 为什么重要

> 反直觉。不是 harness 越"聪明"越好。找到**最小的有效干预点**比构建完整的流水线更重要。

## 10. Epicure — 食物的嵌入空间

### 核心思想

从零开始在 **414 万条食谱**上训练一个多语言配料嵌入族，来自 11 个来源、7 种语言，通过 LLM 增强流水线将原始配料字符串归一化为 **1,790 个规范条目**。

### 三种变体

| 变体 | 训练数据 |
|------|---------|
| Skip-gram (recipe) | 仅食谱共现 |
| Skip-gram (chemical) | 仅 FlavorDB 化合物结构 |
| Skip-gram (blend) | 两者混合 |

### 为什么重要

> 一个紧凑、可下载的食物几何嵌入图谱，是表示学习远远超越文本、延伸到日常领域的一个干净展示。
