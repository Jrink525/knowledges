# 自我改进循环：300 Agent Swarm 完整操作手册

> **来源：** [The Self-Improving Loop](https://x.com/0xmovez/status/2067291911468044494) — Movez (@0xmovez)

![cover](../image/self-improving-loop-kimi-swarm-cover.jpg)

一个免费的开源模型正在用**300 个并行 agent** 在**4,000 步协调**下从单条 prompt 运行，它在真实研究任务上的得分比那些贵 5 倍的模型还高。

大多数人从未打开过 Kimi 的真正能力。他们打开它，输入问题，拿到答案，关掉标签页。这是聊天框模式。它有用，但也只发挥了产品大约 **10%** 的能力。

> 大多数人忽略的核心其实是：

Swarm 不只是跑得快。跑对了，它每次运行都会留下有价值的东西——一个可复用的技能、一份更精准的规范、一条防止下次再犯同样错误的约束。

昨天跑你任务的 swarm 应该比今天跑的 swarm 更聪明。

**这就是循环。** Kimi 负责干活和学习。Opus 4.8 守一道门——验证门——它唯一的任务就是防止垃圾被保存成技能。**引擎学习，守门人保持诚实。**

有人只用一个模型就结婚。有人追着基准排行榜跑。有人配置 LangGraph 花一个周末调试 DAG。结果通常一样：第 50 次运行和第 1 次做的事情一模一样。

本文不是那样。这是 swarm 持续累积成长的操作手册。**10 步。每条 prompt 都可复制粘贴。每个数字都经过验证。**

---

## 01. 写一份 Spec（规范），不是 prompt

大部分人听到「300 agent」就甩出一行字——「研究一下健身应用市场」——然后期待奇迹。这是最快烧光额度拿到垃圾的方式。

一行 prompt 等于授权 swarm 决定一切，而它会决定错。

**把 swarm 当承包商，不是神灯精灵。** Spec 定义了：收集什么、什么算有效、允许哪些来源、确切的输出格式、冲突时怎么办。

大多数人不了解的部分：**Kimi 自己决定如何分解任务。**

你不需要像在 CrewAI 里那样构建 agent，不需要像 LangGraph 那样画图，不需要像 AutoGen 那样定义结构。你描述目标——swarm 自己构建组织架构图。

Spec 是整个循环中最高杠杆率的产物——因为第 4 步它会成为可复用技能的种子。

---

## 02. 在花一分钱之前，先读分解计划

这是新手最常跳过、也最贵的一步。

提交 spec 后，Kimi 在执行前会展示执行计划——多少子 agent、各自处理什么、依赖顺序、步数预算。

**读它。** 一个被错误分解的 200 agent swarm 要花真实的钱和真实的时间。检查计划不要钱。

你要看三件事：
- 它理解范围吗？
- agent 数量对任务大小合理吗？
- 输出计划匹配你的实际需求吗？

值得了解的一个细节：**4,000 步是 swarm 总协调预算**，不是每个 agent 跑 4,000 步。300 agent 的运行平均每人约 **13 步**——短而专注的子任务。这告诉你你的任务是否符合这种形态。

一行 prompt 是一个愿望。一份 spec 是一份订单。Swarm 执行订单。

---

## 03. 放手让它浪费——这正是意义所在

现在开跑。最多 **300 个子 agent** 以并行波次启动。第一波处理完全独立的子任务。结果返回后，编排器(Orchestrator)启动依赖它们的下一波，直到依赖图全部解析完成。

每个子 agent 在自己的**有界上下文窗口**中工作。这是结构上的核心技巧：一个单 agent 长时间执行时会填满窗口直到淹死，然后开始有损压缩，之后每步推理都在变差。

Swarm 给每个子任务独立的上下文范围，所以只有结构化输出流回协调器。**这就是为什么它在单个 agent 容易崩溃的任务上不会崩。**

因为 Kimi 定价 $0.95/M in 和 $4.00/M out——缓存命中时仅 $0.16——你可以**负担把第一次结果扔掉重跑**。便宜到改变你愿意尝试的边界。

---

## 04. 要真正的文件，不要聊天框回答

Swarm 的输出不是窗口里的文本。它是可以直接进入你工作的**结构化制品**——而大多数文章忽略了这部分。

一次运行落地 PDF、电子表格、数据集、PPT 和可运行代码——全部来自一次启动——因为 Kimi **原生生成这些格式**。

> 始终以输出引领 spec。

「一份全面的报告」给 agent 过早停下来的许可。「一份 40 页 PDF + 一个 20,000 行的 CSV + 14 张可导出 PNG 图表」给了它们一个要达到的质量目标。

输出层面的具体性是分水岭。

---

## 05. 用诚实的模型审查输出，指出问题

这是唯一不属于 Kimi 的一步。Swarm 的已知缺陷：除非你明确要求验证，它会产出自信但不加引用的声明，独立的子 agent 有时互相矛盾。「看起来完成了」和「是正确的」是两回事。

Opus 4.8 正是为此设计。Anthropic 报告它比 4.7 **大约少 4 倍的概率**放过自己代码中的缺陷，并且它是第一个在**不批判地报告有缺陷结果上得 0% 分**的 Claude。

它唯一的任务是**反驳，而不是赞美**。你不是花 premium token 来生成——你是花它们来**在第 4 步把缺陷存为永久技能之前抓住它**。

**便宜的体量只有在有值得信任的人在检查时才是一种超能力。** 守住验证门。

---

## 06. 把整个工作流保存为 Skill

这是让循环自我改进的关键。在一次你以后再用的运行之后，告诉 Kimi 把整个工作流捕获为可复用 Skill——输入格式、agent 步骤、输出格式。

**第一次运行 20 分钟。之后每次运行 30 秒。**

这就是「自我学习」的诚实版本。模型不是在你的两次运行之间重新训练权重。**围绕它的系统**在变聪明——你的技能库随着每个项目增长，每个未来的 swarm 自动应用这些技能。

竞争对手不可能在一周内复制那个技能库。它来自你几个月真实运行的积累。

---

## 07. 把你的文档作为 Swarm Knowledge 喂进去

Skill 捕获流程。**文档到 Skill 捕获领域。** 上传你最好的作品——一份成交的方案、一份精良的报告、一个 deck——Kimi 就会捕获它的结构和风格指纹作为 Skill，所有未来的 swarm 自动应用。

这就是复利的来源：每个你喂进去的 PDF、转录稿或电子表格，都成为所有 300 个并行 agent 可以**基于的真实语境**，而不是回归到通用训练数据。

你喂得越多，每次运行的准确性就越高。报告不再看起来像通用 AI 生成——它们看起来像**你的作品**。

---

## 08. 把验证反馈变成永久规则

第 5 步抓住一次缺陷。第 8 步确保 swarm 永远不再犯。

拿 Opus 的修复列表，不要只是修补输出——**把教训写进一个项目级约束文件**，Kimi 在每次会话开始时会自动读取。

这就是循环从自己的失败中学习。Opus 在运行 #1 标识的偏离，变成运行 #2 的硬规则。

经过几个项目后，你的约束文件变成了**自我执行的活文档**——验证门要抓的东西越来越少。

---

## 09. 在新输入上重放 Skill——看成本暴跌

现在回报来了。运行 #2 不是从零开始。它从你在第 6–8 步构建的 **Skill、Swarm Knowledge 和约束文件**开始。

同样的工作流。新的文件。设置时间趋近于零。

这就是「复利」不再是流行语而是在发票上显示出来的地方。第一个竞品监测跑要完整的 spec 和验证通过。第四次是**30 秒 prompt vs 已保存的 skill**，输出因为继承了之前每次运行的修复而更加精准。

**第一次 20 分钟。第五十次 30 秒。** 这个差距就是构建循环而不是构建 prompt 的全部理由。

---

## 10. 把循环升级为后台 Agent

最后一步：一旦一个循环稳定且有 skill 支撑，你就不再需要手动启动它。

把 Kimi 指向触发器——一个定时、一个新文件上传、一个竞争对手的定价页面——让它主动运行整个循环，只汇报产出物和偏离项。

> **竞品监测是最干净的例子。**

运行 #1 你手动构建和验证。到它成为后台 agent 时，它在每周并行检查每个竞争对手，以**零边际时间成本**把简报投到你的收件箱。

循环中剩下的唯一人类，是你提出的问题和你在答案上做的决策。

---

## 结论

当闭源实验室还在每次发一个更聪明的聊天机器人的时候，一个开源模型已经在**并行跑 300 个 agent**——并且随着每次你给它的运行，在系统层面变得越来越聪明。

这个指纹我们已经见过一次了。一次开源发布重新定义了我们以为闭源前沿拥有的东西，整个领域一夜之间重新校准。DeepSeek 就是这样。

**一个能自我学习的 swarm 跑在开源权重模型上，有同样的形态。**

还在争论哪个模型「赢了」的建筑者，是在回答一个已经不重要的问题。

现在的问题不是哪个模型最聪明。而是：
- **你能同时跑多少个？**
- **谁在检查它们的工作？**
- **你的系统今天比昨天更聪明了吗？**

大多数人读完后仍然把 Kimi 当聊天框用。少数人这周就去构建这个循环。

第一次运行 20 分钟。之后每次运行都是你自己的杠杆。

**构建它。验证它。蒸馏它。然后看着它每次运行都变得更加便宜、更加精准。**
