# 构建一个让笔记真正产生价值的 Obsidian 系统 (+ Claude 集成)

> 大多数笔记从未被使用。不是因为你捕捉的信息没有价值，而是因为**捕捉和使用是两个完全不同的活动**——而几乎所有笔记系统都在优化前者，完全忽略了后者。
>
> 这篇指南中的 Obsidian 系统从相反的方向设计：不问"我怎么抓住这个信息"，而是问**"我会怎么使用这个信息"**。

![Cover](../image/obsidian-claude-note-system-1.jpg)

---

## 一、核心理念：捕捉与使用之间的鸿沟

**捕捉**之所以容易是因为没有风险。你划线、快速记一笔、保存到一个文件夹里，感觉很有产出——信息安全了。完了。

**使用**需要更难的东西：在需要的时刻主动检索特定信息，与你已知的内容综合，并转化为**决策、文章、行动、框架**。

大多数笔记系统为**收集**而设计——文件、文件夹、标签、搜索，全部优化在捕捉时刻。这套系统为**综合**而设计——它围绕一个问题构建：**"我应该从已知的内容中产出什么？"**

---

## 二、笔记的四种用途

笔记被"使用"的定义：它贡献了以下**四种结果之一**：

| 用途 | 说明 |
|------|------|
| 🎯 **决策支持** | 你做决定时它浮现并提供相关背景/证据/视角 |
| ✍️ **写作燃料** | 它为你写的内容提供观点、证据、结构 |
| 🗣️ **对话素材** | 在会议/演讲中给你一个本来没有的具体见解 |
| ⚡ **行动触发器** | 在**行动是可能的时刻**浮现想法，而非捕捉的随机时刻 |

> 📌 **准入标准：** 每个笔记至少能贡献于以上一种用途。不能贡献的，就不该在你的 vault 里。

## 三、系统架构：三区结构

```
VAULT/
├── 00 - CAPTURE/         ← 原始素材，尚未加工
│   └── [原始捕获]
│
├── 01 - ACTIVE/          ← 加工后即时有用
│   ├── permanent/        [用自己的话写的原子笔记]
│   ├── projects/         [当前项目]
│   ├── decisions/        [进行中的决策]
│   └── references/       [需定期检索的信息]
│
├── 02 - ARCHIVE/         ← 潜在有用，索引保存
│   ├── projects/
│   ├── decisions/
│   └── references/
│
├── 03 - OUTPUT/          ← ⭐ 最终产出（多数系统没有）
│   └── writing/
│
└── 04 - SYSTEM/
    ├── CLAUDE.md
    ├── skills/
    └── templates/
```

**Output 文件夹是点睛之笔。** 完成的作品——文章、报告、决策、框架——放在这里。它创造了笔记和产出的**可视化联系**，久而久之告诉你哪些笔记是"产出型"的，哪些从未被使用。

## 四、CLAUDE.md：让 Claude 知道"有用"对你意味着什么

```markdown
# 笔记系统 — CLAUDE.md

## 我用笔记做什么
主要用途：[写作/决策/工作/研究]
次要用途：[...]
我产出的内容：[你用知识创造什么]

## 我的活跃项目
[列出笔记应喂入的活跃项目]

## 当前进行中的决策
[列出你正在做的决策]
[这些是笔记现在就该回答的问题]

## 我常写的话题
[列出你定期写的话题]
[处理笔记时，联系到这些话题]

## 什么对我有用
有用的笔记：[具体描述]
无用的笔记：[具体描述]

## 处理标准
笔记准备好进入 Zone 2 当：
- 用我自己的话写，不是从来源复制
- 连接到 Zone 2 中至少一个其他笔记
- 能贡献给至少一个活跃项目或决策
- 标题描述的是想法，不是来源

笔记应归档当：
- 相关项目或决策已完成
- 信息已过时
- 6 个月未被访问

## 产出目标
[你想从笔记中产出什么]
[具体："每周一篇 AI 主题文章"或"每月团队决策回顾"]
```

## 五、五个将笔记转化为产出的工作流

### 工作流 1：日常处理流程（每日）

每晚处理 Zone 1 中积累的笔记。

**Claude 提示词：**

```
处理 00 - CAPTURE 中今天的所有笔记。

对每条捕获笔记：

1. 有用性评估
   这条笔记能否贡献给：
   - CLAUDE.md 中列出的任何活跃项目？
   - 任何活跃决策？
   - 任何写作话题？
   是→标记 PROCESS | 否→直接归档

2. 对标记 PROCESS 的笔记
   用自己的话重写（不是来源摘要，是我的理解 + 我的语气）
   然后识别：
   - 它与哪些现有永久笔记相关？
   - 它喂入哪个活跃项目/决策？
   - 最可能的未来使用场景？

3. 创建永久笔记
   标题：想法本身，不是来源
   内容：我自己的理解
   链接：连接到现有永久笔记
   标签：相关活跃项目和话题

4. 归档原始捕获 → 02 - ARCHIVE

5. 报告处理了什么、什么直接归档了、建立了什么连接
```

### 工作流 2：决策主动喂料器

当你打开一个决策笔记或问问题时，扫描全库寻找相关信息。

**提示词：**

```
我正在做这个决策：[决策描述]
扫描 Zone 2 中的所有笔记。
找到每条相关的笔记：
- 写出笔记标题和关联内容
- 说明它支持、挑战还是增添了决策的维度
然后综合成一份决策简报。
不要从库外添加信息，只综合已有的笔记。
```

> 这是系统的核心原则：你当前问题的答案可能已经在你的旧笔记里了。Claude 跨越它们并浮出水面。

### 工作流 3：写作激活器

每次写作前触发。

**提示词：**

```
我将要写关于：[话题/标题]
扫描 Zone 2 中的所有内容。
生成一份写作简报：
- 最强论点：我的笔记支持的最有辩护力的主张
- 证据：提供证据的特定笔记（附标题）
- 反论点：挑战或复杂化论点的地方
- 具体细节：任何统计数据、例子、引用
- 空白：笔记缺少的、需要研究的内容
- 结构建议：基于笔记内容的推荐结构

不要建议我写什么。告诉我我的笔记已经知道了什么。
```

> 从这个简报出发写作与从零开始写作截然不同。你在**阐述自己已知的**，而不是实时生成想法。

### 工作流 4：连接浮出层（每周）

每周扫描过去 7 天创建的所有笔记，找到与旧笔记的连接。

**提示词：**

```
读取过去 7 天创建或修改的所有永久笔记。
对每条新笔记，扫描整个 Zone 2 库。

有意义的连接包括：
- 相同底层原理应用于不同领域
- 值得一起审视的矛盾主张
- 一条笔记为另一条的主张提供证据
- 跨多条笔记出现的模式

找到每条连接：
- 给出两条笔记的标题
- 描述它们之间的具体连接
- 说明为什么连接它们会使两条笔记更有用

只浮出真正非显而易见的连接。跳过已链接的。
```

> 这就是让笔记库"复利增长"的工作流。你没有刻意创造的连接变得可见。

### 工作流 5：产出生成器

当完成一篇写作、决策总结或项目报告准备好产出时。

**提示词：**

```
我需要产出：[产出类型和描述]
读取 Zone 2 中标记了[项目/话题]的所有笔记。
同时读取连接层已关联到此话题的所有笔记。

生成一份完整草稿：
- 只使用 vault 中的信息
- 用 CLAUDE.md 中描述的我的语气写
- 跨笔记综合而非逐条总结
- 产出一个单条笔记没有、但笔记组合支持的洞察

保存到 03 - OUTPUT/[文件夹]/[日期]-[标题].md
```

> 产出生成器是整个系统证明自己的时刻。数月积累的笔记在几分钟而非几小时内综合为成品。

## 六、让笔记可用的捕获惯例

### 连接捕获

```
[想法或信息]
连接到：[这让你想起什么]
可能用于：[第一个想到的使用场景]
```

多花 30 秒，**价值翻倍**——保留了捕捉时的关联上下文。

### 问题捕获

笔记引发问题时，显式捕获问题：

```
[信息]
这引发了问题：[这个问题产生的疑问]
```

在知识系统中，问题往往比答案更有用。

### 应用捕获

笔记有即时应用时：

```
[想法]
可应用于：[当前工作中的具体场景]
如果正确就：[你会做什么]
```

## 七、每周笔记审计

每周一次，检查 Zone 2 中**14 天未被访问**的笔记：

1. 能否贡献给活跃项目/决策？ → 在相关项目文件夹创建连接笔记
2. 是否连接到任何活跃产出？ → 加链接
3. 什么都不活跃 → 打 `REVIEW` 标签

`REVIEW` 标签笔记如果在连续两次审计后仍无连接 → 归档。

> 审计执行核心原则：**不与任何活跃内容连接的笔记，当前不有用。** 它们属于归档，直到某天重新相关。

## 八、90 天后的效果

| 时间 | 发生什么 |
|------|---------|
| **第 1 周** | 结构化 vault + 5 个工作流 + CLAUDE.md，但笔记积累不够，连接层还无法产生惊喜 |
| **第 2 月** | 连接层开始产生真正有用的链接——不同话题的笔记以你未意识的方式相连 |
| **第 3 月** | **决策喂料器变成最有价值的工作流**——做决策时 Claude 发现 3 个月的 8 条相关笔记，你自己绝不会手动检索到 |
| **第 6 月** | Output 文件夹积累了足够的成品，你能清晰看到哪些笔记是产出型的、哪些从未被用 |

> 系统不只是积累知识。它在**学习什么样的知识对你有用**。这种校准，就是区分"复利增长的系统"和"只会膨胀的系统"的关键。

---

## 九、唯一重要的指标

大多数人用**笔记数量**衡量笔记系统。

**唯一重要的指标：一条笔记被使用的次数。**

> 500 条笔记，每条都贡献给了一个决策、一篇文章或一个行动 → 远胜于 5000 条从未被再次使用的笔记。

构建这套系统，你会发现：
- 写得更快
- 决策更好
- 在你本来会捕捉并遗忘的知识上**采取行动**

这就是笔记系统的真正目的。
