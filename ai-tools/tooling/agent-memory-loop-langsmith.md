## 核心问题

构建一个能从前序行为中学习的循环，让 Agent 产生"记忆"。记忆是用户不需要反复纠正同一个问题的关键——Agent 在第一次被告知后就能记住正确的做法。

## 什么是记忆？

**记忆是持久化的上下文**，Agent 可以在不同运行之间检索它来指导自己的行为。包括：事实、偏好、过去交互、指令、技能、示例和学到的模式。

**关键区分：** 轨迹/日志 ≠ 记忆。轨迹只是发生了什么事的证据。只有当相关的经验被转化为 Agent 能在后续运行中检索并使用的上下文时，才叫记忆。

### 两种作用域

| 类型 | 范围 | 内容 |
|------|------|------|
| **短期记忆（工作记忆）** | 当前任务期间 | 当前线程、最近消息、工具结果、检索文档、中间推理产物、临时状态 |
| **长期记忆** | 跨越多次运行 | 事实、偏好、示例、工作流、策略、指令、技能 |

两者的关系是**读写循环**：
- 运行时：Agent 受益于长期记忆（通过提示组装、检索、工具、文件等方式加载）
- 运行后：轨迹给出证据 → 大部分是历史 → 少量有用信号被提取 → 更新到长期记忆

### 三种长期记忆（认知科学分类）

| 类型 | 定义 | 示例 |
|------|------|------|
| **语义记忆（Semantic）** | Agent 知道的 | 事实、偏好、通用知识 |
| **情景记忆（Episodic）** | Agent 经历过的 | 过去的交互、示例、行为和结果 |
| **程序记忆（Procedural）** | Agent 该怎么做 | 指令、工作流、策略、技能、工具使用规则 |

**最大的改进往往来自程序记忆。** 当 Agent 反复格式错误、工具调用顺序不对、忽略语气规则——修复往往是程序性的：让规则更清晰、改变步骤、把行为移到专门的 Skill 里。

## 记忆循环的三步过程

```
捕获轨迹 → 分析轨迹 → 更新记忆 → 下一次运行加载更新后的上下文
                                 ↑_____________（闭环）_______________|
```

### 第 1 步：捕获轨迹（Traces）

轨迹是证据层。一个良好仪表化的轨迹记录 Agent 走过了什么路径：用户输入、模型调用、工具输入和输出、检索文档、路由决策、延迟、错误、用户反馈。

**为什么重要：** 与传统确定性软件不同，你往往不知道 Agent 表现得怎么样直到你检查了它的轨迹。意外的行为可能是由弱提示词、缺失工具、混乱的工具 schema、差检索、过期上下文、过宽的指令或路由决策引起的。

### 第 2 步：分析轨迹（Analyze）

从轨迹中找到有用的信号：
- 明确的用户反馈或评估失败
- 重复出现的模式：同样的错误输出、同样的无效工具调用、同样的路由错误、同样的被忽略的指令

**棘手的部分：诊断。** 同样的症状可能指向不同的修复。如果 Agent 忽略了语气规则，规则可能太模糊、放错位置、缺少在相关 Skill 中、或与另一条指令矛盾。

### 第 3 步：更新记忆（Update）

系统需要决定未来上下文是否需要改变。可能是修复问题（澄清指令、改变路由规则），也可能是记住有用的东西（用户偏好、成功示例、Agent 应该复用的模式）。

## 用 LangSmith 实现完整循环

LangSmith 的三个产品正好对应三步：

| 步骤 | LangSmith 组件 |
|------|----------------|
| 捕获轨迹 | **LangSmith Observability** — 轨迹存储 |
| 分析轨迹 | **LangSmith Engine** — 分析轨迹的进程 |
| 更新记忆 | **LangSmith Context Hub** — 版本化的记忆存储 |

### LangSmith Engine 的作用

Engine 是一个后台进程，将轨迹转化为改进信号。自动分析轨迹中的重复问题、诊断可能根因、提出具体变更，例如：
- 添加一条规则
- 把某条指令移到更靠近相关工作流的位置
- 创建新的示例
- 更新某个 Skill
- 修改路由策略

### LangSmith Context Hub 的作用

版本化的上下文管理。指令、工具和 Skill 可以在这里管理，确保记忆不是临时拼凑在应用代码里的提示词修改。

## 三条设计原则（经验之谈）

### 1. 不是所有东西都应该变成记忆更新

大多数轨迹数据应该保持为历史记录。有些应该变成数据集示例、评估用例、代码修改或工具 schema 修复。**只有一小部分应该成为持久化上下文。**

### 2. 确保未来运行真的读取了更新

如果运行时缓存了提示词、工具或 Skill，记忆提交后需要有刷新路径。否则系统可能存了正确的更新，但继续用过期上下文运行。

### 3. 用评估保护重要行为

如果一个记忆更新足够重要来塑造未来行为，通常值得设置一个方式来检测该行为是否退化。

---

## 一句话总结

> **轨迹是证据，记忆是经验。Engine 提取信号，Context Hub 让信号持久化。每次运行都是下一次的改进起点。**
