## 为什么读这篇？

每天观察人们用 Claude 的**吃力方式**：
- 重复写相同指令
- 每次重新解释上下文
- 手动在应用间复制粘贴
- 每次提示词不一样，结果不稳定

**普通用户和高级用户之间的差距不是天赋，不是经验，不是付费计划。而是 Skills 和工作流。**

Skills = 教会 Claude 如何执行特定任务的永久指令文件。  
Workflows = 可重复的系统，消除猜测工作，每次产出一致结果。

大多数新用户甚至不知道它们存在。已经用上的人，产出量是普通用户的 **三倍**，时间只用 **一半**。

## Part 1: 你还没安装的十大 Essential Skills（1–10）

### 01. PDF Processing Skill（PDF 处理）
读取 PDF、提取表格、填写表单、合并拆分文档。不仅是"读文件"——是**完整结构化提取**，含表格解析和批量处理。对任何处理文档的人来说，这是最高价值的 Skill。

### 02. DOCX Skill（Word 文档）
生成带**正确格式、修订痕迹、批注、专业排版**的 Word 文档。不是 Claude 聊天框输出的纯文本——而是含页眉页码的 `.docx` 成品文件。省去"复制文本到 Word"的中间步骤。

### 03. PPTX Skill（演示文稿）
从自然语言描述自动生成幻灯片：排版、图表、演讲者备注、统一设计。"给我创建一份 12 页的初创公司 Pitch Deck"——直接拿到 PowerPoint 文件。

### 04. XLSX Skill（电子表格）
用纯英语指令创建**带公式、分析、格式和图表**的电子表格。"构建一个 12 个月的收入预测财务模型"——拿到一个带真实公式的可工作表格。

### 05. Frontend Design Skill（前端设计）
打破 AI 通用"毛坯房"审美。教 Claude 真正的设计系统——大胆排版、独特配色、专业布局。**超过 27.7 万次安装**自有其原因。只要你在做任何 UI，这个 Skill 不可跳过。

### 06. Canvas Design Skill（画布设计）
社交媒体图片、海报、封面、视觉内容——从文本描述生成，输出 PNG/PDF。对大多数任务来说，这个 Skill 能替代独立设计工具。

### 07. SEO Skill（SEO 优化）
完整网站审计、Schema 验证、关键词分析、内容优化——**12 个子技能**覆盖 SEO 全流程。不同于问 Claude 泛泛的 SEO 建议，这个 Skill 执行系统化审计，给出具体可行的发现。

### 08. Brand Guidelines Skill（品牌指南）
把你的品牌——颜色、字体、语气、信息规则——编码成一个 Skill，自动应用到所有场景。每个文档、每个设计、每段内容都遵循品牌规范，无需每次提醒。

### 09. Marketing Skills Bundle（营销技能包）
**20+ Skills 打包**，覆盖 CRO、文案、SEO、邮件序列、增长策略。一次安装，Claude 获得整个营销领域的专业知识。

### 10. Obsidian Skills（Obsidian 笔记管理）
由 Obsidian CEO 亲自构建。自动标签、链接、整理你的 Obsidian 知识库。把 Claude 变成你的**笔记库管家**。

## Part 2: 每周省数小时的 Workflows（11–20）

Skills 教 Claude **怎么做**。Workflows 教 Claude **什么时候做什么及顺序**。

### 11. The Morning Briefing Workflow（晨间简报）
设一次，每天早晨跑：
> "检查邮件是否有紧急事项。汇总昨夜 Slack 消息。拉今日日历。识别今天最重要的三件事。保存到每日简报文件。"

设成 7am 定时任务。**起床即收到完整战报。**

### 12. The Content Production Pipeline（内容生产流水线）
四步走：研究主题 → 创建大纲（标题/章节/要点） → 写完整草稿 → 编辑润色达标。在同一 Cowork session 中跑完所有步骤。**一篇完整文章不到 30 分钟。**

### 13. The Weekly Report Compiler（周报编译器）
每周五 5pm：
> "拉本周所有完成任务。汇总 [来源] 的关键指标。编译胜利项、阻塞项、下周优先级，保存为格式化报告。"

**周报在你收工的同时自己写好了。**

### 14. The Email Response Workflow（邮件回复）
每封需要回复的邮件：
> "读此邮件。识别发件人的请求。起草专业、五句以内、含明确下一步的回复。匹配发件人的正式程度。"

**20 封邮件 = 平时写 3 封的时间。**

### 15. The Competitive Intelligence Workflow（竞争情报）
每月定期：
> "研究 [竞争对手名单]。检查新品发布、定价变化、功能更新、公开声明。对比我方位置。保存情报简报。"

无需手动研究数小时，自动掌握竞争动态。

### 16. The File Organization Workflow（文件整理）
每周：
> "扫描 Downloads 文件夹。按类型归类文件。统一命名规范。删除 60 天以上未标记为重要的内容。"

**数字工作区自动保持整洁。**

### 17. The Client Onboarding Workflow（客户入职）
每个新客户：
> "创建 [客户名] 文件夹。用模板生成欢迎文档。基于 [项目范围] 创建时间线。起草入职邮件。全部保存到客户文件夹。"

**几分钟完成专业入职体验，而非几小时。**

### 18. The Learning Summary Workflow（学习总结）
消费任何教育内容后：
> "读我的 [主题] 笔记。提取核心概念。识别每项概念如何应用于当前工作。创建一页带行动项的总结。保存到 /Learning。"

**每本书、每门课、每篇文章 -> 可行动的知识。**

### 19. The Social Media Scheduling Workflow（社交媒体排期）
每周：
> "基于内容日历，为下周每天起草各平台帖子。按平台格式：Twitter ≤280 字符，LinkedIn ≤200 词，Instagram 带话题。按天分文件夹保存。"

**一次 session 备好整周社媒内容。**

### 20. The Invoice and Receipt Workflow（发票收据处理）
每月：
> "处理 /Receipts 中所有收据图片。提取日期、商家、金额、类别。创建按类别排序的电子表格。计算总额。标记异常项。保存到 /Finance。"

**每月报销从 1 小时 -> 几分钟。**

## Part 3: 高级用户配置（21–30）

以下设置区分**普通用户**和**高手**。

### 21. The Master Context File（主语境文件）
创建 `context.md`，包含 Claude 需要知道的关于你的一切：
- 你是谁、做什么
- 你的受众在意什么
- 语气和沟通风格
- 当前项目和优先级
- 你覆盖的主题和避免的主题
- 最佳作品示例

在每个 Skill 和 Workflow 开头加上 `"Read context.md fully before starting any task"`。**这个文件消除了每次对话重新解释的麻烦。**

### 22. The CLAUDE.md File for Code Projects（代码项目配置）
使用 Claude Code？在每个项目根目录创建 `CLAUDE.md`，包含：技术栈、编码规范、目录结构、偏好。

Claude 每次 session 前读取并自动遵循。

> **经验法则：** 当同一件事纠正 Claude 两次以上，就写入 CLAUDE.md。文件会越来越聪明。

### 23. The Prompt Library（提示词库）
创建 `/Prompts` 文件夹，把表现最好的提示词存为独立文件。不再从零写提示——引用你的保存库。

### 24. The Template System（模板系统）
为每个重复产出建模板：报告模板、邮件模板、文档模板、幻灯片模板。上传到项目知识文件，需要时直接引用："用我的报告模板，填本周数据。"

### 25. The Evaluation Checklist（质量检查清单）
让 Claude 在交付输出前自检：
```
交付前验证：
- 完全遵循指令吗？
- 格式和长度正确吗？
- 所有声明是否具体（数字、名字、日期）而非模糊？
- 语气符合要求吗？
- 我能不编辑就发给客户吗？
如有任何"否"，修复后再给出。
```
**把这个加在每个 Skill 末尾，质量马上提升。**

### 26. The Error Recovery Protocol（错误修复协议）
Claude 犯错时，普通用户当场纠正就过去了。**高手更新自己的 Skill：**
- "加条规则：营销文案永远不用被动语态。"
- "加条规则：引用统计数据时必须包含来源 URL。"
- "加条规则：引用任何时效信息前先确认当前日期。"

**每个错误变成永久修复。**

### 27. The Multi-Project Workspace（多项目管理）
为每个主要工作领域设置独立 Claude Project。每个项目有自己的 System Prompt、知识文件、对话历史。

**上下文干净，输出相关。**

### 28. The Connector Stack（工具连接栈）
连接每个日常应用：Gmail、Google Calendar、Slack、Google Drive、Notion。

**Claude 能直接访问的越多，你在工具和 AI 之间当传话筒的时间越少。**

### 29. The Feedback Loop System（反馈闭环）
每周五花 15 分钟回顾 Claude 本周产出：
- 哪个输出没达标？
- 什么指令改变能修复它？
- 下周能加什么新任务？

基于答案更新 Skills 和 Workflows。**这是大多数用户完全遗漏的复利效应。**

### 30. The Skill Builder Workflow（Skill 构建器）
用 Skill Creator meta-skill 按需建新 Skill。描述你手动做的重复工作，让 Claude 把它转成永久 Skill：
> "我目前每周手动做 X。把它转成 Claude 能自动执行的 Skill，质量保持一致。"

**每个重复任务 = 一个自动化 Skill。你的系统每周变得更强大。**

## 官方视角：Anthropic Skills 架构详解

来自官方文档的关键补充：

### Skills 体系结构

| 层级 | 路径 | 适用范围 |
|------|------|----------|
| 企业级 | 见 managed settings | 组织全部用户 |
| 个人级 | `~/.claude/skills/<name>/SKILL.md` | 你的所有项目 |
| 项目级 | `.claude/skills/<name>/SKILL.md` | 仅当前项目 |
| 插件级 | `<plugin>/skills/<name>/SKILL.md` | 插件启用处 |

### 与 Claude Code 的关系

Anthropic 官方将 Skills 定位为 Claude Code 的扩展机制。和 Khairallah 文中所述一致：

- **Skills = `.claude/skills/<name>/SKILL.md`**，以目录+SKILL.md 为入口点
- **CLAUDE.md = 事实与偏好**（Khairallah 的 #22），Skill = 多步流程（#1-10）
- **内置技能（Bundled skills）** 包括 `/code-review`, `/debug`, `/run`, `/verify`, `/loop` 等
- **实时热加载**：增删改 Skill 文件后**不需重启**，当前 session 即时生效

### Skills vs Commands（重要更新）

> 自定义命令已合并入 Skills 体系。`.claude/commands/deploy.md` 和 `.claude/skills/deploy/SKILL.md` 都能创建 `/deploy` 命令，效果一致。Skills 有额外特性：支持文件目录、Frontmatter 调用控制。

### Frontmatter 高级配置（补充 Khairallah 文中未提及的能力）

```yaml
---
name: my-skill
description: 这是我的技能
context: fork          # 在子 agent 中运行，不消耗主对话的上下文
allowed-tools: Read Grep  # 限制该技能只能使用特定工具
disable-model-invocation: true  # 只有我能调用，Claude 不会自动触发
---
```

### 动态上下文注入

Skills 支持 `!`\`command\``` 语法——Claude Code 在执行技能前运行指定命令并把输出注入到提示词中。

```
## 当前变更

!`git diff HEAD`
```

这让 Skills 基于**实时数据**工作，而非刻板模板。

## 与 "Claude Code Hooks" 的对照

Khairallah 这篇和 [上次整理的 vincemask Hooks 文章](./claude-code-hooks-automation-workflow.md) 形成**互补关系**：

| 维度 | Khairallah 30 Workflows | Vincemask Hooks |
|------|------------------------|-----------------|
| 用户群体 | 所有 Claude 用户（含网页端 + Claude Code） | Claude Code 开发者 |
| 核心机制 | Skills + Workflows + Project 配置 | `.claude/rules.md` + Hooks 事件系统 |
| 场景重心 | 文档处理、内容生产、业务运营 | 代码开发、Git 工作流、测试验证 |
| 哲学 | 让 Claude 成为日常工作搭档 | 让 Claude 成为代码开发全流程助理 |
| 复用方式 | Skills 可跨项目共享 | Hooks 搭配 `--rules-mode` 精准绑定 |

**结合起来看：**
- **日常生产力** → Khairallah 的 Workflows（晨间简报、内容流水线、邮件处理）
- **开发工作流** → vincemask 的 Hooks（pre-commit 检查、测试、代码规范）
- **基础底座** → 两者都强调的 CLAUDE.md（#22） + Master Context 文件（#21）
