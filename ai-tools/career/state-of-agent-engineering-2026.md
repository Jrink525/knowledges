## 一、Agent 的定义：明确边界

本报告统一的 Agent 定义 —— **能设定目标、规划多步工作、调用工具（API、数据库、内部系统）完成任务，且能基于中间结果自主决策下一步行动**。

**分类层级：**
- **Chatbot**：对话式界面，无可靠工具执行能力
- **Copilot**：嵌入工具（IDE、邮件、文档）的辅助系统，自动化有限
- **Tool-calling Agent**：能调用工具/API，执行多步计划，带护栏
- **Multi-agent System**：多 Agent 分工协作，通常通过编排器协调

**什么不算 Agent：** 无工具调用的纯聊天界面、无状态的单步 prompt-in/text-out 自动化、营销意义上的"Agent"。

## 二、LangChain 报告核心发现（1,340 份答卷）

### 2.1 生产化势头强劲

51% → 57.3%（同比去年）。大多数组织的问题已从"要不要做 Agent"变成"怎么可靠落地"。

| 公司规模 | 已投产 | 积极开发中 |
|----------|--------|-----------|
| 10,000+ 人 | 67% | 24% |
| <100 人 | 50% | 36% |

大企业落地更快，可能得益于平台团队、安全基础设施和可靠性投入。

### 2.2 主要用例

| 用例 | 占比 | 说明 |
|------|------|------|
| 客服 | 26.5% | Agent 直面客户，不再只是内部工具 |
| 研究 & 数据分析 | 24.4% | 信息综合、跨源推理 |
| 内部工作流自动化 | 18% | 提高员工效率 |
| 代码生成 | — | 高频日常使用，但非单选项 |

**大规模企业（10,000+ 人）的差异：** 内部生产力（26.8%）> 客服（24.7%）> 研究分析（22.2%）

**开发者日常最常用的 Agent：**
1. **编码 Agent** 最多（Claude Code、Cursor、Copilot、Windsurf 等）
2. **研究型 Agent** 其次（ChatGPT Deep Research、Claude、Gemini、Perplexity）
3. **自定义 Agent**（基于 LangChain/LangGraph 搭建的内部 QA、知识库、SQL 等）

### 2.3 最大生产障碍

| 障碍 | 占比 | 说明 |
|------|------|------|
| **质量** | 32% | 准确率、一致性、品牌合规（与去年一致，仍为第一）|
| **延迟** | 20% | 多步 Agent 质量高但慢，用户体验受影响 |
| **成本** | 下降 | 模型降价，优先级降低 |
| **安全** | 24.9%（大企业） | 2000+ 人企业第二大担忧 |

对于万人以上企业，写明回复指出 **幻觉和输出一致性** 是质量最大挑战，其次是 **上下文工程和规模化上下文管理**。

### 2.4 可观测性：标配

| 指标 | 占比 |
|------|------|
| 已实施某种形式可观测性 | **89%** |
| 全链路追踪（逐步骤、逐工具调用） | **62%** |
| 已投产者具备可观测性 | **94%** |
| 已投产者具备全链路追踪 | **71.5%** |

可观测性已超过评估（eval）成为 Base Level 工程实践。

### 2.5 评估与测试

| 评估方式 | 占比 |
|----------|------|
| 离线评估 | **52.4%** |
| 在线评估 | **37.3%** |
| 结合离线和在线 | ~25%（在有评估的组织中）|
| 完全不评估 | 29.5%→22.8%（已投产的降低）|

**评估方法偏好：**
- 人工审查：**59.8%** — 高价值场景仍不可替代
- LLM-as-Judge：**53.3%** — 规模化评估质量/事实准确性
- ROUGE/BLEU：**采用率有限** — 不适合开放式 Agent 交互

### 2.6 模型生态

| 指标 | 数据 |
|------|------|
| 使用 OpenAI GPT 模型 | >67% |
| 使用多种模型 | >75% |
| 运行自己部署的开源模型 | ~33% |
| 不微调 | 57% |

模型多样化是常态，精调仍是"特战武器"而非标准化流程。

## 三、Lexogrine 交叉验证：各大报告对比

### 3.1 核心问题：各家报告在测量什么？

最大的陷阱是**各家的"Agent"定义和衡量尺度不同**：

| 来源 | 测量类型 | 样本 | 关键数字 |
|------|---------|------|---------|
| **Microsoft** | 平台遥测（硬数据） | Fortune 500 + 全量用户 | >80% Fortune 500 有活跃 Agent |
| **McKinsey** | 自报告调查 | 1,993 人，105 国 | 23% 规模化，39% 实验 |
| **ServiceNow / Oxford** | 高管自报告 | ~4,500 高管 | 33% 试点或有可用案例 |
| **Deloitte** | 高管自报告 | 3,235 人，24 国 | 74% 计划两年内，21% 有治理 |
| **KPMG** | 高管自报告 | 2,500 人，27 国 | 88% 在投资 Agentic AI |
| **CrewAI** | 厂商赞助高管调查 | 500 C-level | 65% 已使用，79% 增加投入 |
| **LangChain** | 构建者调查 | 1,340 人 | 57.3% 投产，30.4% 开发中 |

**要点：** 不要把这些数字直接混在一起当统一指标。

### 3.2 按维度拆解

#### 按公司规模

| 规模 | 投产率 | 来源 |
|------|--------|------|
| 10,000+ 人 | 67% | LangChain |

大企业领先，但差距没有想象中大。大企业的领先可能来自平台团队和安全基础设施，而非技术能力。

#### 按地区（Microsoft 生态内）

| 地区 | 活跃 Agent 占比 |
|------|----------------|
| EMEA | 42% |
| 美国 | 29% |
| 亚洲 | 19% |
| 美洲（不含美国）| 10% |

注意：这是微软生态内的活跃 Agent 分布，非整体渗透率。

#### 按行业（Microsoft 生态内）

| 行业 | 活跃 Agent 占比 |
|------|----------------|
| 软件与技术 | 16% |
| 制造业 | 13% |

#### 按投资意愿（KPMG）

88% 高管表示正在投资将 Agentic AI 构建进系统——覆盖八个主要行业。

### 3.3 实验到生产的鸿沟

Deloitte 指出：仅 **25%** 的组织将 40% 或以上的 AI 实验转化为生产。这是很好的对标基准——如果感觉自己的转化率低，你并不孤单。

### 3.4 生产化的核心阻碍（多方验证）

| 阻碍 | 数据来源 |
|------|---------|
| **质量**（准确率/幻觉/一致性） | LangChain 32%、Deloitte |
| **延迟** | LangChain 20% |
| **安全/治理滞后** | Deloitte: 21% 有成熟治理；Microsoft: 47% 应用安全控制 |
| **影子 IT** | Microsoft: 29% 员工使用未授权的 AI Agent |
| **系统集成难** | Salesforce: 96% 认为 Agent 成功依赖跨系统数据打通 |

## 五、趋势总结与工程建议

### 5.1 不可忽视的趋势

1. **Agent 已不是"要不要做"，而是"怎么做"**。57% 已投产，加上 30% 正在开发——总计近 90% 的组织正在进行中
2. **质量 > 成本**。成本担忧下降，质量（幻觉、一致性、延迟）成为第一杀手
3. **可观测性已是 Base Level**，评估（eval）是下一个主战场
4. **多模型是常态，精调仍是少数派**
5. **治理严重滞后**。29% 员工已在使用未授权的 Agent，影子 AI 风险不容忽视
6. **客服与数据分析是最大用例，编码 Agent 是开发者日常主力**
7. **微软生态内 Agent 在实地运行，Fortune 500 超 80% 已有活跃部署**

### 5.2 工程建议

- **第一天就上可观测性**：89% 的人已经做了，没做的在拖后腿
- **质量优先于成本**：评估→调试→迭代→再评估，这是 Agent 工程的核心循环
- **大企业先做安全/治理**：否则影子 IT 会先替你部署
- **重视延迟**：到用户的 Agent，20% 受访者已经卡在延迟上
- **不要押注单一模型**：75%+ 团队已用多模型，按复杂度/成本/延迟路由

## 六、关键提醒：数据怎么读

Lexogrine 给出了很重要的方法论提醒：

**"AI 使用" ≠ "Agent 使用"** —— 多数工具的"AI 功能"仍停留在 Chatbot 层面

**"实验" ≠ "生产"** —— 各家的定义完全不同：
- McKinsey 的"规模化" ≠ LangChain 的"投产" ≠ Microsoft 的"活跃 Agent"
- 跨报告对比时务必明确每个数字的具体含义

**调查 vs 遥测：**
- 遥测（Microsoft 家）→ 硬数据，但限单一生态
- 调查（McKinsey/Deloitte/KPMG）→ 覆盖广，但自报告偏差大
- 构建者调查（LangChain/CrewAI）→ 反映前沿，但样本有偏
