# Hermes Sensei Loop：用冠军循环让研究 Agent 自我进化

> **来源：** [The Hermes Sensei Loop](https://x.com/0xjeff/status/2069008589721932251) — @0xJeff

![cover](../image/hermes-sensei-loop-1.jpg)

**你给 AI 一个 prompt，AI 吐出一个输出。这是生成式 AI。**

**你给 AI 一个 prompt，AI 吐出输出，从中学习，主动提议下一步，设置定时 cron 任务每天自动交付。这是 Agentic AI，也是 Hermes 真正擅长的事。**

但有一个问题——输出/cron 的质量取决于模型 + 工具 + prompt + cron 格式。你需要主动和 agent 对话，它才能改进 cron 质量。

如果你像我一样主要用 Hermes 做研究，还会遇到 AI 的"谄媚"问题——不是那种觉得你主意最好的谄媚，而是像 23 岁日本打工人一样拼命鞠躬、对你并不好笑的笑话陪笑的那种谄媚。没有对抗性审查，没有论点压力测试，cron 们还要靠你来给反馈才改进。

> **这就是 AI agent "循环"登场的地方。**

不再是单次 prompt 产生单次回答，模型进入一个**思考→行动→观察→重复**的循环，把 agent 向自主问题解决者推进一步。

---

## Hermes 已有的循环

Hermes 已经有类似循环的机制：agent 接受用户的 prompt/反馈，从中学习，下次做得更好。这是一个**依赖人类反馈的循环**。

而我们的目标是把人类反馈替换成 **agent 自己的反馈**，节省时间、提高效率。

**大多数人把循环跑在编程 agent 上。我把循环跑在研究 agent 上。**

### 3 个核心目标

1. 提升主要研究 cron 任务的质量
2. 验证/反驳/压力测试论点，决定是加注还是转向
3. 让 agent 更自主，学会自己思考

## 1. 自我改进冠军循环（Self-Improving Champion Loop）

冠军循环解决了**输出质量停滞**的问题。它把 Hermes 的自我学习能力再推一步——创建一个每天增量改进输出质量的循环。

**以前的方法是看心情给反馈：** "太随大流了"、"格式太乱了"。有时改进了，有时没有。

**冠军循环把 prompt 改进从"凭感觉"变成"经得起验证的改进"。** 每个变更必须在从未见过的数据上证明自己。

### 工作原理

你有一个**冠军（当前 prompt）**和一个**留存集（holdout set——6 天未经处理的输出+人工标注的 ground truth）。** 每个挑战者必须在留存集上以明确的提升幅度击败冠军——同时不违反任何硬性规则。

如果做不到，就被拒绝，冠军保留。这防止了 ad-hoc 调 prompt 的两个失败模式：**过拟合**（只在你盯着看的那几个例子上变好）和**无声回归**（修好一个，打破了另一个你没注意的）。

![champion loop diagram](../image/hermes-sensei-loop-2.png)

### 四步流程

**Step 1: 冻结基线**
拿你当前的 prompt（"冠军"），在 20 个历史例子上评分。14 个用来修改，6 个绝不碰——留存集。记录两边的分数。

**Step 2: 只修一件事**
选一个记录在案的具体失败。修改 prompt 只修这个问题——不多动任何东西。这是你的"挑战者"。

**Step 3: 用未见数据证明**
先在工作集上测挑战者。看起来好了？冻结，然后在留存集上测——那 6 个你从未对着修改过的例子。挑战者只有在留存集上以明确幅度击败冠军、且不违反任何硬性规则时，才能晋级。

**Step 4: 改善停止就收手**
达到目标分数 → 发布。预算用完 → 发布最好的冠军。连续两轮在留存集上没有提升 → 到局部最优了，停止修改。

> **让冠军循环起效的唯一规则：绝不在工作集上晋级。**
>
> 如果在工作集上晋级，就会过拟合——prompt 在你盯着看的例子上变得很好，在新例子上变差。留存集晋级是免疫系统。它在发布前捕捉回归。

**就像训练初级分析师：** 你给他 14 份交易备忘录练习。他学会模式——你标记什么、忽略什么、怎么组织推荐。14 份后，每份都能完美应对。

然后你让他上活交易。

第 15 份，他从未见过的——他自信地推荐对一个周期性矿业公司做 4x 杠杆 LBO，因为"备忘录 #4 就是这么做的"。但备忘录 #4 是家 90% 经常性收入的 SaaS 公司。他学到的模式无法泛化。他记住了例子，没有理解原则。**过拟合。**

这就是为什么你不在构建策略的同一数据上回测。

**这个循环的好处：**
- 每一轮变更都被记录、验证、打分、测试
- 输出质量随时间持续显著提升

## 2. 反馈清扫循环（Feedback Sweep Loop）

这是向冠军循环输送养料的**自动收集器**。

它监听你抱怨了什么，按工作流组织，然后把"下一步修什么、按抱怨次数排序"的干净列表交给冠军循环。

## 0xJeff 的当前工作流

他把这两个循环应用到三个核心工作流：

| 工作流 | 说明 |
|--------|------|
| **Equities Daily** | 股票组合每日报告——关注什么、顺风/逆风、待探索的新板块 |
| **Top 7 Synthesis** | 当天 Top 7 头条，与过往上下文的变动，为什么重要，行动项（买入/持有/卖出） |
| **Alpha Triage** | 按层级排序当前对投资组合最重要的事 |

> 冠军循环要跑起来，需要先为每个工作流收集工作集和留存集。0xJeff 正在提供反馈、评分和偏好，希望一周后能开始自动运行。

---

## 为什么费这么大劲

让 agent 离自主存在更近一步，需要把**推理、执行和反馈循环**都委托给 agent 本身。创建并启动这些循环是第一步。

做这件事还学会了一件事：**给反馈时要更有条理、更结构化。** 提供结构化反馈而不是模糊的意见。说话/打字前先想好。某种程度上，因为 agent 我变成了更好的管理者 lol。
