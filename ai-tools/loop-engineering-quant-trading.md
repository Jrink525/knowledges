# 如何用 Loop Engineering 构建自改进的量化交易系统

> **来源：** [How To Use Loop Engineering To Build A Self-Improving Quant Trading System](https://x.com/rohonchain/status/2069056530960490835)
> **作者：** Roan (@RohOnChain)，后端开发者，专注于系统设计、HFT 型执行引擎和量化交易系统

## 引言

大多数量化交易者还在"提示" Claude。他们打字。等待。阅读输出。再打字。

而地球上最聪明的构建者已经不再这么做了。

他们**写循环**。循环让 Claude 做事。循环验证输出。循环决定下一步。循环在合上笔记本之后继续运行。

Anthropic Claude Code 负责人 Boris Cherny 两周前说得很直白：「我不再提示 Claude 了。我有循环在运行，它们提示 Claude 并决定下一步做什么。我的工作就是写循环。」这一句话重新定义了地球上每一个严肃 AI 工程师思考构建的方式。而且它完美契合量化交易。

大多数散户量化交易者会读到这里说这不适用于他们，因为他们太小了。他们错了。**你的资金越少，这个就越重要。** 一个自动运行的循环，是独立构建者缩小与拥有 100 个 PhD 的基金之间差距的唯一路径。

因为量化交易本身就是一个循环：拉取数据 → 生成信号 → 回测 → 执行 → 监控风险 → 重复。

华尔街上每一家基金都在运行这个精确的循环。文艺复兴自 1988 年就开始这样跑。Citadel 依靠工程师团队监控每个阶段。Two Sigma、Jane Street 都一样。

唯一的区别是：他们需要几百个人坐在循环里面。**你不需要。**

我已经为自己构建了这个循环。它按计划拉取市场数据。它运行 alpha 研究。它通过一个独立的 agent 验证每个信号。它只执行通过验证的信号。它把每一个教训写入记忆。

这篇文章是我学到的关于 loop engineering 的一切，以及如何将其接入一个完整的自主交易系统。

读完本文，你将了解：

- 提示一个 agent 和工程化一个循环之间的确切区别
- 生产中每个工作循环的六块积木
- 如何将这六块组合成一个从零开始的自改进量化交易系统

## Part 1：提示与 Loop Engineering 的区别

过去两年，用 AI 工作是这样的：你打字提示 → 你阅读返回的内容 → 你根据看到的内容输入下一个提示。

**你就是那个循环。**

Agent 只是一个工具。你全程握着它。每一步都是你坐在键盘前决定下一步做什么。

Loop Engineering 终结了这一点。

**你不再是循环里面的东西。你变成了设计架构的人。**

一个循环是一个递归目标。你定义目的。Agent 围绕它迭代。循环持续运行直到真实的终止条件满足。

Agent 在运行之间会忘记。**循环不会。**

这一个事实就是整个架构。

这就是 Boris 说他工作是写循环时的意思。他不再一条一条地输入指令。他构建的系统替他发出指令、读取结果、决定下一步。

对编码来说，这改变了软件交付的方式。

对交易来说，这改变了一切。

因为没有哪个量化交易者是通过一次输入提示然后走开就赚到钱的。**优势来自把同一个循环运行几千次，每次改进 1%，并且永不睡觉。**

这正是循环做的事。

如果你还在一个交易一个交易地往 Claude 里敲提示，你就在做 Boris 两年前就不做的事了。杠杆已经上移了一层。你不再是写更好的提示了。**你在写发出提示的系统。**

## Part 2：每个工作循环的六块积木

一个工作循环由六部分组成。缺一个，循环就悄悄崩溃。

### 1. 自动化（Automation）

这是心跳。一个 cron 调度、一个 webhook、一个 `/loop` 命令，或者 Claude Code 内部的一个钩子，不需要你打字就能触发。

有两种值得了解的形式：
- **`/loop`**：按固定节奏运行，不关心状态
- **`/goal`**：持续运行直到你写的可验证条件真正为真，由另一个小型模型来判断工作是否完成

在交易中，`/loop` 是你每隔一分钟拉取数据。`/goal` 是「持续迭代这个信号直到回测 Sharpe 比率超过 1.5」。

### 2. 技能（Skill）

技能是一本 procedure manual，agent 读它而不是每次从头被告知。

它存在于一个 SKILL.md 文件里。它包含你的约定、你的规则、你的「我们因为那次事故而不这么做」。

没有技能，每个循环运行都从零开始。有技能，意图会累积。

### 3. 状态文件（State File）

一个 markdown 文件，通常叫 STATE.md 或 PROGRESS.md。

它在运行之间持续存在。Agent 会忘记。**文件不会。**

Agent 在每次运行开始时读取它。在结束时写回发生了什么。

这听起来太简单以至于不像回事。但实际上它是每个工作循环的**脊梁**。

### 4. 验证器（Verifier）

写代码的 agent 是判断代码是否正确的最差人选。

应用到交易：生成信号的 agent 是判断信号是真正的 alpha 还是噪声的最差人选。

你需要一个**独立的 agent**，用不同的指令，理想情况下是不同的模型，它唯一的工作就是验证产出。

这就是 maker-checker 模式。华尔街上每个自营交易公司内部都是这么组织的。在 Jane Street，提出交易的交易员不批准交易。在 Citadel，构建模型的研究员不验证模型。

### 5. 工作树（Worktrees）

一旦你在同一个文件上运行超过一个 agent，它们就开始冲突。

Git worktrees 给每个 agent 自己独立的工作目录，指向自己的分支。

在交易中，这让你并行运行信号研究、回测和风险监控，而互不干扰。

### 6. 连接器（Connectors）

一个只能读文件的循环是个微小的循环。

基于 Model Context Protocol (MCP) 的连接器让循环可以调用经纪商 API、查询数据库、发消息到 Slack、向交易所发送订单。

这就是建议交易的循环和**实际下单**的循环之间的区别。

这六块积木是通用的。它们出现在 Claude Code 里。出现在 Codex 里。出现在地球上每个工作的 agentic 系统中。

## Part 3：如何构建自改进的量化交易循环

量化交易循环有五个阶段。每个阶段是一个自己的子循环，有自己的技能、自己的状态文件、自己的验证器。

### 阶段一：数据摄入

自动化根据资产类别每分钟、每小时或每天触发一次。

数据进入一个共享状态文件，下一个阶段读取。

### 阶段二：信号生成

这里是 alpha 研究发生的地方。

信号生成 agent 读取一个 SKILL.md 文件，其中包含你的 alpha 研究规则。

**技能文件随时间增长。每个亏损写回一条新教训。每条教训成为下次运行的新规则。**

这就是系统自改进的方式。

### 阶段三：验证

信号被发送到一个**完全独立的 agent**。不同的模型。不同的指令。不接触原始信号是如何推演的。

如果验证失败，信号被杀死。如果通过，进入执行。

验证器**永远不看**信号生成器推理了什么。这个分离就是整个优势。

你也可以用更强的模型做 checker，较弱的模型做 maker。Claude Opus 验证，Claude Sonnet 生成。不同的模型架构捕获不同类型的错误。这和机器学习中使用集成方法的逻辑相同。

### 阶段四：执行

只有经过验证的信号到达这一阶段。

MCP 连接器处理经纪商 API。循环从不请求权限。自动模式让它无需人工干预即可运行。

### 阶段五：风险监控

全程在并行 worktree 中运行。

这是**终止开关**。它强制执行规则，没有任何协商余地。

---

这五个子循环组成一个自动运行的系统：

> 数据流入 → 信号生成 → 信号验证 → 已验证信号执行 → 风险监控 → 教训写回记忆 → 然后重新开始

我设计了一次。之后再没有人工提示过这些步骤中的任何一步。

这就是 loop engineering。这就是 Boris 说他工作是写循环时的意思。

## 一个警告

没有真正终止条件的循环会安静地失败。Agent 发出完成信号，认为半完成的工作已经完成。循环退出。坏交易敞开着。

**你的终止条件需要由 agent 自身声明之外的东西来检验：**
- "过去 30 笔交易中 Sharpe 比率高于 1.5"
- "回撤低于 5%"
- "测试套件通过"

**永远不要用「agent 说它完成了」。**

## 总结

量化交易本身就是一个循环。华尔街上每个基金都在跑。他们只是需要几百个人坐在循环里。

Loop Engineering 移除了人。

**六块积木**组成每个工作循环：
- **自动化**触发心跳
- **技能**持有项目知识
- **状态文件**持有记忆
- **验证器**给产出打分
- **工作树**隔离并行工作
- **连接器**让循环在现实世界中拥有手

把它们围绕五个阶段交易周期组装起来，你就有了一个**自改进的系统**，它自主运行 alpha 研究、验证信号、执行交易和监控风险。

系统每一轮都在变得更聪明。每个亏损写出新教训。每个教训变成新规则。一百笔交易之后，技能文件是一份活文档。一千笔之后，它已经接近机构知识，比任何一个人脑能记住的都要多。

**最先构建这个的基金会为下一个十年积累优势还在提示的那些会被抛在后面。**

所以，这里的问题是：

如果 loop engineering 是提示之上的下一层抽象，而量化交易是世界上赌注最高的循环——
**你是在一个交易一个交易地敲提示的人，还是设计了那个在你睡觉时替你交易的循环的架构师？**

没有错误的答案，但有一些极具揭示性的答案。
