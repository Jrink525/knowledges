## 1. 背景：从个人到组织的 Agent 进化

在 Anthropic 内部，**Claude Tag**（在 Slack 中 @Claude 来调用的 Agent）经历了迅猛增长。Lance 加入 Anthropic 时，几乎每个人都有自己的 Claude Code 操作平台（harness），带有个人专属的连接器 —— Agent 的身份是**绑定在个人**身上的。

但过去几个月，**Claude Tag 引入了一个组织级的 Agent 操作平台**（org-level agent harness），在 Slack 中赋予 Claude 属于自己的身份、连接器和技能。这一转变不只是部署方式的改变，更是 Agent 协作范式的根本性重构。

> **核心洞察**：Claude 是光源，产品是聚焦它的透镜。随着 Claude 越来越强大，正确的产品透镜也随之变化。

---

## 2. 三大行为转变

Lance 将 Claude Tag 带来的变化归纳为三个关键维度：

### 2.1 单人→多人协作

**个人操作平台的局限：**
- 每个人有自己的 Claude Code + 连接器
- 新员工需要从头搭建自己的 Agent 环境
- Agent 的知识和技能是孤立的碎片

**组织级操作平台的优势：**

| 优势 | 说明 |
|------|------|
| **贡献平权** | 新员工和老员工拥有**相同的操作平台**，极大缩短 onboarding 时间 |
| **共享上下文** | Claude 可以访问公共频道（在定义的安全边界内），减少重复劳动，传播最佳实践 |
| **内联协作** | Claude "嵌入"团队工作流，而非独立运行 |

**实际协作模式：**
- 事故发布到 Slack 频道，**@Claude** 帮助多名工程师在共享线程中排查根因
- 在长篇讨论线程中 **@Claude**，让它撰写包含时间线和行动项的故障复盘报告

### 2.2 同步→异步工作

**传统 Slackbot 的痛点：**
- Claude 只能处理有限的自主工作
- 用户需要不断 check-in、督促、审批
- 模型能力不支持流畅的异步 Agent UX

**Claude Tag 的异步新范式：**

Lance 的精辟比喻：
> **Claude 是光源，产品是聚焦它的透镜。** 随着 Claude 越来越能胜任长时间任务，正确的产品透镜从"你盯着它"变成了"它盯着任务"。

Claude 的任务长度每 4 个月翻一番，这使得 Claude Tag 成为越来越有吸引力的产品表面。用户不再需要坐在终端前盯着 Claude Code 跑 —— 你把任务交给 @Claude，它自己跑，跑完了回来找你。

**异步工作与 Goal 原语的配合：**
```text
"找到改进延迟的机会，检查结果，做性能分析"
"把框架 X→Y 迁移到整个代码库，逐项验证端到端正确性"
```
Claude 运行 → 对照目标衡量 → 自我修正 —— 形成一个完整的闭环。

### 2.3 被动响应→主动出击

**异步 UX 的内在需求：** Claude 需要来找你，而不是你去找它。

如果 Claude 跑了 6 小时卡住了，用户希望收到**主动告警**。但主动性偏好是任务相关的、也是用户相关的。Claude Tag 通过**频道级记忆**（channel-based memory）使其高度可配置。

**记忆系统是关键解锁：** Claude 能捕获并记住你对主动性的偏好设置。

**三种主动性模式的实践：**

| 模式 | 说明 | 配置方式 |
|------|------|----------|
| 始终响应 | 频道内任何消息都回应 | 频道记忆：always respond |
| 按需响应 | 只回应特定请求类型 | 频道记忆：respond to specific request types |
| 事件触发 | 按计划或事件驱动 | 设置 schedule 或 event-driven triggers |

## 3. 技术支撑

Claude Tag 的成功并非单一因素的产物，而是多项技术能力汇聚的结果：

### 3.1 模型能力：任务长度每 4 个月翻倍

Claude 的任务处理能力（task length）每隔四个月翻一番。这意味着：
- 从只能处理几分钟的任务 → 可以运行数小时的自主工作
- 从需要用户实时监督 → 可以放心委托后做别的事
- Claude Tag 作为产品表面，随着模型能力的增强而越发具吸引力

### 3.2 安全体系：可信任的异步委托

异步工作委托的核心前提是**信任**。三个安全层面的进展：

1. **抗提示注入增强**（Prompt injection resistance）
   — Claude 更难被恶意提示劫持

2. **自动模式分类器**（Auto-mode classifiers）
   — 在操作执行前对动作进行审核

3. **Claude Tag 的安全模型**
   — 为异步委托构建了强大的安全框架

### 3.3 新原语：Goal 自驱动循环

Goal 原语让 Claude 能够：
1. 接收高层次目标（如"优化延迟"）
2. **自主运行**任务代码
3. **对照目标衡量**结果
4. **自我修正**偏差

这构成了 Agent 的**闭环自驱系统**，不再需要人类在每一步介入。

### 3.4 沟通层：HTML 作为 Agent 表达媒介

长时间运行的 Agent 需要一种丰富的输出形式。Lance 和同事 trq212 探讨了 **HTML 作为 Claude 的沟通表面**：

- 实验或数据科学工作的结果，Claude 能生成**自定义 HTML 仪表盘**
- 包含可视化图表、交互式元素
- 让大量实验数据以人类可读的结构化方式呈现

## 4. 与 Claude Code CLI 的互补关系

Claude Tag **不是要取代 Claude Code CLI**，而是互为补充：

| 维度 | Claude Tag (Slack) | Claude Code (CLI) |
|------|-------------------|-------------------|
| 交互模式 | 异步、被动/主动 | 同步、主动操作 |
| 典型场景 | 发起任务、监控、告警 | 深度编码、调试 |
| 协作粒度 | 团队级、频道级 | 个人级、文件级 |
| 使用方式 | @Claude + 自然语言描述 | 终端命令 + 代码交互 |

**Lance 的实际工作流：**
1. 用 Claude Tag **发起工作**（在 Slack 中 @Claude）
2. 对于一些需要更亲手上手的任务，**到 CLI 跟进**（Claude Code）

这种"分层的 Agent 表面"策略，让用户在不同场景下选择最合适的交互方式。

## 5. 实用场景与最佳实践

### 5.1 故障排查

**场景**：事故告警发到 Slack 频道  
**操作**：@Claude 加入共享线程  
**效果**：Claude 与多名工程师协作排查根因，提供上下文、历史数据和可能的修复方案

### 5.2 事后复盘

**场景**：长篇讨论线程中讨论了一次故障  
**操作**：@Claude 写入 postmortem  
**效果**：Claude 自动梳理时间线、提取关键事件、生成行动项清单

### 5.3 实验监控

**场景**：长时间运行的实验  
**操作**：告诉 Claude "监控这个实验"  
**效果**：Claude 持续观察实验状态，当实验可能失败或需要干预时**主动通知**用户

### 5.4 主动搜索

**场景**：开始新任务前需要了解历史上下文  
**操作**：@claude 主动搜索频道历史  
**效果**：Claude 自动检索相关频道中的过往工作，让你在开始前就了解最佳实践和已知陷阱

### 5.5 频道守望者

**场景**：想在特定条件满足时被通知  
**操作**："ping me if <某个条件满足>"  
**效果**：Claude 持续观察频道，条件触发时 @提及用户

### 5.6 代码迁移

**场景**：框架升级（如 X→Y）  
**操作**：给 Claude 一个 Goal："Migrate framework X→Y across the code"  
**效果**：Claude 遍历代码库执行迁移，并验证每个迁移的端到端正确性

### 5.7 性能优化

**场景**：系统延迟需要优化  
**操作**：给 Claude 一个 Goal："Find opportunities to improve latency, check results, and profile"  
**效果**：Claude 自主寻找优化点、实施改进、验证效果

## 7. 总结与展望

Lance Martin 的这篇长文揭示了一个明确的趋势：**Agent 正在从个人工具进化为组织基础设施**。

### 关键 takeaways

1. **操作平台的层级进化**：个人操作平台 → 组织级操作平台。Claude Tag 不是简单的 Slack 集成，而是一个**拥有独立身份、连接器和技能的 Agent**。

2. **交互模式的三个转变**：
   - 单人 → 多人（贡献平权、共享上下文、内联协作）
   - 同步 → 异步（长期任务委托、Goal 驱动的自我闭环）
   - 被动 → 主动（频道记忆驱动的个性化主动性）

3. **多面性互补**：Claude Tag (Slack) 和 Claude Code (CLI) 是互补而非替代关系，用户根据任务类型选择最合适的 Agent 表面。

4. **Slack 只是开始**：Lance 明确指出 "Slack 只是第一个表面"（the first of several surfaces），暗示将来 Claude Tag 会扩展到更多平台。

5. **安全是基石**：异步委托的核心前提是可信任的安全模型，抗提示注入 + 自动分类审核 + 细粒度权限控制缺一不可。

### 对行业的意义

当一位 Anthropic MTS 专门撰文分享内部如何用 Agent 时，这标志着：

- **Agent 操作平台**（agent harness）正在成为组织级基础设施的核心组件
- **"AI 原生协作"** 不再是理论概念，而是在 Anthropic 内部被实际验证的工作方式
- 组织级 Agent 的记忆系统（channel-based memory）为个性化、配置化 Agent 提供了关键基础
