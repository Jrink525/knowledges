### Bugs Found & Fixed During Session

1. **Sync cleanup bug**: First run of _organize.py deleted local-only files (the 4 new Java/Spring docs) because `sync_from_github()` had a cleanup step that removed local files not on remote.
   - **Fix**: Removed the local-only file cleanup step from `sync_from_github()` — only download remote changes, never delete local.

2. **Classification priority bug**: `interview`/`面试` tags were matching too early, causing Java and Spring docs to be classified into `interview/` instead of `programming/java/` or `spring/`.
   - **Fix**: Reordered CLASSIFICATION so programming languages and framework rules come before generic `interview`/`高并发` rules.

3. **Missing nested dir support in CLASSIFICATION**: `programming/java/` path broke the delete-empty-directories logic because `existing_dirs` only contains top-level dirs (like `programming/`) but `classified_dirs` has full paths (like `programming/java`).
   - **Fix**: Added `used_prefixes` tracking — if a top-level dir is a prefix of any classified dir (e.g., `programming` is prefix of `programming/java`), don't delete it.

4. **File deleted before move**: In `sync_classified_to_local()`, when a file changed categories (e.g., from `database/` to `spring/`), the cleanup step in the old directory ran *before* the move, deleting the source file. Then the move failed with FileNotFoundError.
   - **Fix**: Split into 3 phases: (1) move all files, (2) cleanup target dirs, (3) delete empty dirs.

## Afternoon Session: Classification System Rewrite + New Knowledge Doc

### Major Change: `_organize.py` Classification Rewrite
- **Replaced** static `CLASSIFICATION` tuple-matching with **dynamic content-based classification**
- New approach: reads full `.md` file content → extracts technical keywords → scores 7 domains → picks highest score
- **Tags are only a bonus (+3/tag)**, not the primary classification mechanism
- **Title keywords get +5 bonus** (stronger signal than body keywords)

### Domain Profiles (7 categories)
- `programming/java/` — Java/JVM/concurrency keywords (synchronized, volatile, GC, AQS, etc.)
- `spring/` — Spring Framework keywords (IoC, AOP, @Transactional, auto-configuration, etc.)
- `database/` — Redis, MySQL, SQL keywords (redis, innodb, 事务, 索引, etc.)
- `sre/` — SRE/operations keywords (SLA, circuit breaker, 限流, 监控, etc.)
- `infrastructure/` — Network/cloud/container keywords (firewall, docker, k8s, openclaw, etc.)
- `system-design/` — Architecture/distributed/dependency keywords (微服务, CAP, 分布式事务, etc.)

### Bug Fix: Tag Matching with Longer Keywords
- **Problem**: `spring` tag couldn't match `spring framework` keyword because the keyword is longer than the tag. `tag.startswith(kw)` failed.
- **Fix**: Added standalone `"spring"` keyword to Spring domain, plus logic checks both directions.
